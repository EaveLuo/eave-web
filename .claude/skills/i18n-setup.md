# Skill: Setup i18n for New Column

When user asks to add a new documentation column with internationalization:

## Steps

1. **Create folder structure** (ALWAYS use English names):
   ```
   docs/{column-name}/
   ├── intro.md
   └── _category_.json
   ```

2. **Configure _category_.json**:
   ```json
   {
     "label": "column-name",
     "position": 1,
     "collapsed": false,
     "link": {
       "type": "generated-index"
     }
   }
   ```

3. **Create Chinese intro** (`docs/{column-name}/intro.md`):
   ```markdown
   ---
   sidebar_position: 1
   keywords:
     - 关键词 1
     - 关键词 2
   ---

   # 栏目名

   该模块主要介绍 XXX 相关的知识，包括...
   ```

4. **Create English intro** (`i18n/en/docusaurus-plugin-content-docs/current/{column-name}/intro.md`):
   ```markdown
   ---
   sidebar_position: 1
   keywords:
     - keyword 1
     - keyword 2
   ---

   # Column Name

   This module mainly introduces...
   ```

5. **Add i18n mappings**:
   - `i18n/zh-CN/current.json`:
     ```json
     {
       "sidebar.{sidebarId}.category.column-name": {
         "message": "中文栏目名",
         "description": "..."
       }
     }
     ```
   - `i18n/en/current.json`:
     ```json
     {
       "sidebar.{sidebarId}.category.column-name": {
         "message": "English Column Name",
         "description": "..."
       }
     }
     ```

6. **Update navigation** (`docusaurus.config.ts`):
   - Add to `navbar.items`
   - Add to `footer.links`

7. **Update sidebars** (`sidebars.ts`):
   ```typescript
   {
     columnSidebar: [{ type: 'autogenerated', dirName: 'column-name' }]
   }
   ```

## Important Rules

- NEVER use Chinese folder names
- ALWAYS create both Chinese and English versions
- ALWAYS add i18n mappings
- NEVER add `sidebar_label` in front matter
- ALWAYS add `keywords` metadata
