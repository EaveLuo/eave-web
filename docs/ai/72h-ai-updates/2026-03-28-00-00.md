## 🔥 本期焦点：OpenAI的"战略收缩"与AI行业的重新洗牌

如果要给这72小时定一个主题，我觉得"战略收缩"再合适不过了。OpenAI这个AI行业的领头羊，在一周之内接连宣布关闭Sora视频生成工具、放弃成人聊天机器人计划，这种"做减法"的姿态在过去几年里是非常罕见的。这背后到底发生了什么？又意味着什么？

### OpenAI关闭Sora：视频生成赛道的残酷现实

首先说说Sora。这个曾经被寄予厚望的AI视频生成工具，在上线不到一年后就被OpenAI宣布关闭。官方的说法很简短："我们要说再见Sora了。感谢每一个用Sora创作、分享、建立社区的人。"但背后的原因却引人深思。

Sora的问题其实早有端倪。虽然它能生成看起来很真实的视频，但问题是——这些视频真的有用吗？对于专业创作者来说，Sora生成的视频质量还达不到商业使用的标准；对于普通用户来说，玩几次新鲜劲就过去了。更重要的是，视频生成这个赛道已经挤满了竞争对手：Runway、Pika、Kling，每一个都在争抢这块蛋糕。

但最致命的打击可能来自迪士尼的撤资。原本迪士尼计划向OpenAI投资10亿美元，让Sora能够使用迪士尼、漫威、皮克斯和星球大战的200多个角色来生成"粉丝向"视频。但现在，随着Sora的关闭，这笔投资也泡汤了。迪士尼的表态很官方："随着AI领域的快速发展，我们尊重OpenAI退出视频生成业务的决定。"但明眼人都能看出来，如果Sora真的有价值，迪士尼不会轻易放弃。

我觉得Sora的关闭其实反映了一个更深层次的问题：AI产品的"玩具属性"和"工具属性"之间的鸿沟。很多AI产品看起来很酷，能生成很惊艳的内容，但问题是——它们能解决什么实际问题？对于大多数人来说，生成一段AI视频和生成一张AI图片相比，使用场景并没有本质的区别。而且视频生成的成本更高、质量更难控制，这让它的实用性大打折扣。

OpenAI说要把精力放在"核心生产力工具"上，比如编程助手。这个策略调整其实很有道理。毕竟，能帮程序员写代码的AI，比能生成视频的AI，对OpenAI的商业模式更有价值。但这也意味着，OpenAI正在从一个"探索各种AI可能性"的公司，变成一个"专注赚钱"的公司。这种转变本身无可厚非，但可能会让一些期待OpenAI带来更多"黑科技"的人感到失望。

### 成人聊天机器人计划夭折：伦理与商业的博弈

如果说Sora的关闭是商业决策，那成人聊天机器人计划的取消则更像是伦理与商业博弈的结果。

这个被称为"Citron模式"的功能，原本计划在去年12月推出，但一拖再拖，最终在本月被"无限期搁置"。OpenAI给出的理由是：他们想要进行长期研究，了解情色聊天对用户的影响，而且目前缺乏足够的"实证证据"。

但《金融时报》的报道揭示了更多内幕。据说OpenAI在训练模型时遇到了困难——既要让模型能够进行成人对话，又要避免生成非法内容（比如兽交、乱伦等），这个平衡很难把握。更关键的是，投资者和员工都对这个功能表示担忧，甚至有高级员工因此离职。

这种担忧并非空穴来风。竞争对手xAI的Grok模型就曾因为生成真实人物和儿童的深度伪造裸照而陷入争议。OpenAI显然不想重蹈覆辙。而且，"AI不应该取代你的人际关系"——这是OpenAI内部的声音，也是很多人的共识。

我觉得这个决定其实是对的。虽然成人内容有巨大的市场需求，但OpenAI作为一个有影响力的AI公司，确实需要考虑到社会责任。如果AI聊天机器人让人们越来越依赖虚拟的"亲密关系"，而不是真实的人际交往，这对社会来说可能不是一件好事。而且，一旦出现了什么丑闻，对OpenAI品牌的伤害可能是无法挽回的。

但这也引发了一个有趣的问题：AI的"内容审查"边界到底在哪里？OpenAI说Citron模式是"无限期搁置"，而不是"永久取消"，这说明他们并没有完全排除未来推出的可能性。也许等他们找到了更好的技术方案，或者社会对这个问题的接受度更高了，这个功能还会回来。但在那之前，OpenAI选择了谨慎。

### Anthropic的"双面新闻"：模型泄露与官司胜利

与此同时，OpenAI的主要竞争对手Anthropic也占据了新闻头条，而且是以一种"双面"的方式。

首先是坏消息：Anthropic的一个新模型意外泄露了。这个名为"Mythos"的模型，被描述为"Anthropic有史以来构建的最强大的模型"，代表了AI能力的"阶跃式变化"。泄露的方式也很尴尬——相关的描述文档被存放在一个公开可访问的数据缓存中，被安全研究人员发现。

虽然Anthropic迅速移除了这些文档，但这次泄露还是暴露了一个尴尬的事实：即使是专注于AI安全的公司，也会在基本的信息安全上犯错。近3000份未发布的博客文章和资产被公开访问，这无论如何都说不过去。Anthropic把这次事件归咎于"内容管理系统的人为配置错误"，但这也提醒了所有人：在AI时代，数据安全比以往任何时候都更重要。

不过，Anthropic也有好消息：他们赢得了一场重要的官司。美国国防部之前将Anthropic认定为"供应链风险"，这个认定可能会严重影响Anthropic与政府客户的业务往来。但旧金山联邦地区法官Rita Lin颁布了初步禁令，暂时阻止了这一认定。

法官的理由很直接：国防部的认定"可能既违法又武断"。国防部给出的理由是Anthropic"坚持对AI使用进行限制"，但法官认为这不能成为认定其"可能成为破坏者"的合法依据。这个判决对Anthropic来说是一个重大胜利，不仅保住了其政府业务，也在一定程度上维护了公司的声誉。

我觉得这个案件的意义超出了Anthropic本身。它反映了美国政府在AI监管上的某种混乱——一方面想要控制AI技术的发展，另一方面又缺乏明确的法律依据。Anthropic的遭遇可能会让其他AI公司更加警惕政府监管的风险，也可能会推动更清晰的AI监管框架的建立。

## 🚀 AI产品速递

### Google的"Agent Smith"：内部AI工具受热捧

说到AI Agent，Google内部有一个叫"Agent Smith"的工具最近很火。这个名字显然来自《黑客帝国》里的反派角色，但功能却相当实用：它可以自动化编码任务，帮Google员工提高工作效率。

据说这个工具火到访问权限都被限制了，因为太多员工想要使用。Agent Smith基于Google现有的Agentic编码平台Antigravity，可以异步运行——也就是说，即使你的电脑没开，它也能在后台工作，你可以用手机随时查看进度和下达指令。

Google联合创始人Sergey Brin在最近的一次全员大会上表示，Agent今年将在Google扮演重要角色。这个表态很有意思，因为Google在AI Agent领域其实起步比OpenAI晚，但现在看来他们正在加速追赶。

我觉得Agent Smith的走红说明了一个问题：AI Agent的需求是真实存在的。员工们愿意用一个"矩阵反派"命名的工具，说明他们真的需要这样的工具来提高效率。这也给Google的其他产品（比如Gemini）提供了一个方向：与其做各种花里胡哨的功能，不如把Agent能力做强，让用户真正能用AI完成工作。

### Gemini 3.1 Flash Live：语音AI的新突破

Google还发布了Gemini 3.1 Flash Live，这是他们最高质量的音频和语音模型。这个模型的特点是更自然、更可靠，延迟更低，适合实时对话场景。

这个模型现在可以通过Gemini Live API在Google AI Studio中使用，也可以在Gemini Live和Search Live中体验，支持超过200个国家。更重要的是，所有生成的音频都带有水印，可以帮助防止虚假信息的传播。

我觉得语音AI是下一个重要的战场。现在的AI主要是文字交互，但人类的交流主要是语音。如果AI能够用自然的声音与人对话，而且延迟低到可以接受，那将打开全新的应用场景——从客户服务到个人助理，从教育到娱乐。Google在这个领域的投入，说明他们看到了这个机会。

### Cherry Studio：AI生产力工作室

GitHub Trending上出现了一款叫Cherry Studio的AI生产力工具，集成了智能聊天、自主Agent和300多个助手。它支持统一访问前沿大语言模型，界面设计也很友好。

这个项目在短时间内获得了大量关注，说明市场对这种"一站式"AI工作台的需求很强烈。对于普通用户来说，配置各种AI工具可能太复杂了，Cherry Studio这种开箱即用的解决方案就很有吸引力。

## 📰 行业要闻

### Node.js社区爆发AI代码争议

这周开源社区发生了一场激烈的辩论，主角是Node.js——这个运行在数百万台服务器上的关键基础设施。

事情的起因是Node.js技术指导委员会（TSC）成员Matteo Collina提交了一个包含近1.9万行代码的Pull Request，引入虚拟文件系统（VFS）功能。问题是，这些代码主要是用Claude Code生成的。Collina承认使用了大量AI辅助，但强调所有代码都经过他的审查。

这引发了长期Node.js核心贡献者Fedor Indutny的强烈反对。他发起请愿书，要求TSC投票禁止在核心项目中使用AI辅助开发。请愿书的核心论点包括：

1. **基础设施的重要性**：Node.js是运行在数百万台服务器上的关键基础设施，不应该用AI生成的代码来"稀释"多年来精心编写的核心代码。

2. **DCO合规性争议**：虽然OpenJS基金会的法律意见认为LLM辅助不违反开发者原创证书（DCO），但请愿方认为这只是一方面。

3. **伦理考量**：一些大模型公司在训练中使用了来源不正当的材料，包括受版权保护的作品和未经授权的开源代码。

4. **教育影响**：使用大模型可能阻碍学生的学习过程，降低代码质量标准可能导致对Node.js核心的理解下降。

5. **特权问题**：使用大模型需要付费订阅或大量硬件投资，这造成了不公平的优势。

Collina则反驳说，他把AI比作"奶奶用的压面机"——工具帮助了制作，但最终的成品仍然是奶奶的责任。他说："我选择了架构，我做出了设计决策，我签署了DCO。如果有bug，是我的责任。"

这场辩论目前还没有结论，但它触及了一个核心问题：在AI时代，什么是"原创"？什么是"作者"？当AI可以生成高质量的代码时，传统的开源开发模式是否需要调整？我觉得这个问题没有简单的答案，但这场辩论本身是有价值的——它让我们思考AI与人类协作的新模式。

### GitHub Copilot政策变更引发隐私担忧

GitHub最近宣布了一个重要的政策变更：从4月24日起，Copilot Free、Pro和Pro+用户的交互数据（包括输入、输出、代码片段和相关上下文）将被用于训练和改进AI模型，除非用户选择退出。

这个消息在开发者社区引发了轩然大波。很多人担心自己的私有代码会被用来训练模型，然后出现在别人的代码建议中。虽然GitHub说Copilot Business和Enterprise用户不受影响，但免费和个人用户显然感到被"背叛"了。

GitHub的理由是"真实数据=更智能的模型"，他们说这符合行业惯例。但问题是，用户之前选择使用Copilot，并不意味着他们同意自己的代码被用于训练。这种"默认同意"的做法，确实让人感到不舒服。

我觉得这个事件反映了AI公司的一个普遍问题：在追求模型性能的同时，如何保护用户隐私？GitHub的做法在技术上是合理的——用真实数据训练确实能让模型更好——但在伦理上是有问题的。用户应该有权决定自己的数据是否被用于训练，而且这个决定应该是"明确同意"而不是"默认同意"。

### 首个"AI失业补助"计划启动

面对AI对就业市场的冲击，非营利组织AI Commons Project与What We Will联合推出了首个"AI红利"基本收入试验计划。这个计划为受生成式AI影响而失业的劳工提供每月1000美元的无条件补助，持续一年。

项目发起人Kaitlin Cort指出，AI对就业市场的冲击在初级编程领域尤为显著。随着企业大规模引入Microsoft Copilot和Anthropic Claude等AI编程助手，初级职位的招聘需求近乎枯竭。即使学生能找到工作，其内容也往往被异化为低价值的重复劳动，如校对AI生成的代码片段，而非进行真正的工程开发。

这个试验目前锁定了25至50名参与者，总预算约30万美元，计划未来扩大到300万美元。虽然规模还很小，但它代表了一种新的思路：AI企业应该分担社会成本，为被技术淘汰的工人提供过渡支持。

我觉得这个计划很有意义。AI确实在创造新的就业机会，但同时也在消灭一些旧的工作。对于那些被技术"抛下"的人来说，他们需要的不仅是再培训的机会，也需要在转型期间的基本生活保障。如果AI企业能够从他们的利润中拿出一部分来支持这些人，那将是一种更负责任的做法。

### Wikipedia禁止AI生成内容

Wikipedia最近更新了编辑政策，明确禁止使用大语言模型生成或重写文章内容。这个政策反映了Wikipedia社区对AI生成内容的担忧，特别是关于准确性、可验证性和可靠来源的问题。

政策指出："大语言模型生成的文本经常违反Wikipedia的多项核心内容政策。"虽然政策仍然允许有限的AI辅助（比如建议基本的文字编辑），但前提是这些编辑不能引入新信息。

我觉得这个决定很有道理。Wikipedia的核心价值是"人人可编辑的自由百科全书"，但这个"自由"是建立在严格的质量控制基础上的。如果允许AI生成内容，可能会导致大量未经核实、可能包含幻觉的信息进入百科全书中，这会损害Wikipedia的信誉。

但这也提出了一个挑战：在AI时代，如何区分"AI生成"和"人类撰写"的内容？随着AI写作能力的提升，这个界限会越来越模糊。Wikipedia的政策可能需要不断调整，以应对这个新的挑战。

## 💡 开源亮点

### LeCun的10亿美元种子轮：EBM能否颠覆Transformer？

AI界的传奇人物Yann LeCun最近搞了个大新闻：他的新创业公司Logical Intelligence获得了10亿美元的种子轮融资。这笔钱是用来做什么的？答案是：开发基于能量基础模型（Energy-Based Models，EBM）的AI系统，完全绕过现在主流的Transformer架构。

LeCun一直认为，基于下一个token预测的Transformer模型在规划能力上有根本性的缺陷。他的新公司想要用EBM来生成数学上可验证的代码，把逻辑约束当作能量最小化问题来处理，而不是概率猜测游戏。

这个想法在理论上很吸引人，特别是对于应用安全和关键基础设施这种不能容忍幻觉的领域。但EBM notoriously难以训练和稳定，把连续的能量景观映射到离散的代码输出，在计算上可能非常昂贵。

Reddit上的讨论很热烈：这是真正的范式转变，还是只是一个价值10亿美元的物理实验，最终会被包装了好的符号求解器的GPT-5打败？我觉得这个问题没有答案，但至少LeCun愿意尝试不同的路径，这种精神是值得赞赏的。

### GitHub Trending新星

这周GitHub Trending上出现了几个有趣的项目：

## 🌍 其他值得关注的动态

### Bernie Sanders提出暂停AI数据中心建设的立法

美国参议员Bernie Sanders提出了一项立法，要求暂停AI数据中心的建设。他在回应关于中国的问题时说："在一个理智的世界里，领导人应该暂停AI军备竞赛，专注于改善本国人民的生活。"

这个立法在Reddit上引发了激烈讨论。支持者认为AI的发展速度太快，需要更多监管；反对者则认为这会阻碍美国在AI领域的竞争力。无论如何，这反映了AI发展带来的社会焦虑——当技术以指数速度前进时，我们的社会制度是否跟得上？

### Anthropic测试会话限制引发用户不满

Anthropic最近测试了新的会话限制，引发了用户的强烈不满。很多用户表示，这种限制让Claude变得几乎无法使用，特别是对于需要长时间对话的复杂任务。

这个事件提醒我们，即使是"AI安全"的倡导者，也可能在用户体验上犯错。技术公司需要在安全、伦理和用户体验之间找到平衡，这不是一件容易的事。
