# 程序结构 - 包、导入与可见性

在 Go 语言中，程序是由**包**（package）组织的。理解包的概念是掌握 Go 语言的第一步。

## 📦 什么是包？

包是 Go 程序的基本组织单元，每个包都有独立的命名空间。

### 包的作用

1. **模块化**：将相关功能组织在一起
2. **命名空间**：避免命名冲突
3. **封装**：控制代码的可见性
4. **复用**：包可以被多个项目共享

### 包的基本结构

```go
// 每个 Go 文件必须以 package 声明开头
package main  // 可执行程序

package fmt   // 库包
```

**规则：**
- `package main`：可执行程序，必须包含 `main()` 函数
- 其他包名：库包，提供可复用的功能

## 👁️ 可见性规则（导出规则）

Go 只有一种可见性控制机制：**首字母大小写**

```go
package utils

// ✅ 导出（首字母大写）
var Config string      // 外部可访问
func Helper() {}       // 外部可调用
type Person struct {}  // 外部可使用

// ❌ 未导出（首字母小写）
var config string      // 仅包内访问
func helper() {}       // 仅包内调用
type person struct {}  // 仅包内使用
```

## 🔗 导入包（import）

```go
import (
    "fmt"                    // 标准库
    "github.com/gin-gonic/gin"  // 第三方库
    myfmt "myproject/utils"  // 别名导入
    _ "database/sql"         // 匿名导入（只执行 init）
)
```

**⚠️ 规则：** 导入的包必须使用，否则编译错误！

## 🔄 包的初始化

```go
// init 函数自动执行（可以有多个）
func init() {
    fmt.Println("init 1")
}

func init() {
    fmt.Println("init 2")
}
```

**初始化顺序：**
1. 导入的包先初始化
2. 包级变量按声明顺序初始化
3. `init()` 函数按声明顺序执行
