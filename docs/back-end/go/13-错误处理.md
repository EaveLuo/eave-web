# 错误处理 - Go 式的优雅

Go 没有异常，只有错误。这种看似简单的方式，却带来了更清晰、更可靠的代码。

## 🎯 错误即值

在 Go 中，错误是一个值：

```go
type error interface {
    Error() string
}

// 函数返回错误
func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("除数不能为零")
    }
    return a / b, nil
}

// 处理错误
result, err := divide(10, 0)
if err != nil {
    fmt.Println("错误:", err)
    return
}
fmt.Println("结果:", result)
```

## 🔧 创建错误

### 1. errors.New

```go
err := errors.New(" Something went wrong")
```

### 2. fmt.Errorf（格式化）

```go
name := "张三"
err := fmt.Errorf("用户 %s 不存在", name)
```

### 3. 自定义错误类型

```go
type ValidationError struct {
    Field   string
    Message string
}

func (e ValidationError) Error() string {
    return fmt.Sprintf("%s: %s", e.Field, e.Message)
}

// 使用
err := ValidationError{
    Field:   "email",
    Message: "格式无效",
}
```

## 🎨 错误处理模式

### 1. 直接返回

```go
func process() error {
    if err := step1(); err != nil {
        return err
    }
    if err := step2(); err != nil {
        return err
    }
    return nil
}
```

### 2. 包装错误

```go
func loadConfig() error {
    data, err := os.Open("config.json")
    if err != nil {
        return fmt.Errorf("打开配置文件：%w", err)
    }
    // ...
}
```

### 3. 忽略错误（谨慎！）

```go
// 明确忽略
_, err := someFunc()
if err != nil {
    // 记录日志但不处理
    log.Printf("警告：%v", err)
}

// 或明确忽略返回值
_ = os.Remove("/tmp/file")
```

## 🔍 错误检查技巧

### 1. 提前返回

```go
// ✅ 清晰
func process() error {
    if err := validate(); err != nil {
        return err
    }
    if err := load(); err != nil {
        return err
    }
    // 主逻辑
    return nil
}

// ❌ 嵌套过深
func process2() error {
    if err := validate(); err == nil {
        if err := load(); err == nil {
            // 主逻辑
        }
    }
    return err
}
```

### 2. 错误辅助函数

```go
func check(err error) {
    if err != nil {
        log.Fatal(err)
    }
}

// 使用
file, err := os.Open("file.txt")
check(err)
```

### 3. 多次错误处理

```go
func process() (err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("panic: %v", r)
        }
    }()
    
    // 可能 panic 的代码
}
```

## 🎁 错误包装和解包

### 1. 包装错误

```go
err := os.Open("file.txt")
if err != nil {
    return fmt.Errorf("读取文件失败：%w", err)
}
```

### 2. 解包错误

```go
err := doSomething()

// 检查具体错误
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("文件不存在")
}

// 提取具体错误
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println("路径错误:", pathErr.Path)
}

// 解包
unwrapped := errors.Unwrap(err)
```

## 🐛 常见错误

### 1. 忽略错误

```go
// ❌ 危险
file, _ := os.Open("file.txt")
// file 可能是 nil！

// ✅ 正确
file, err := os.Open("file.txt")
if err != nil {
    // 处理错误
}
```

### 2. 错误信息不清晰

```go
// ❌ 模糊
if err != nil {
    return errors.New("failed")
}

// ✅ 清晰
if err != nil {
    return fmt.Errorf("加载用户%d的数据：%w", userID, err)
}
```

### 3. 重复包装

```go
// ❌ 过度包装
err := step1()
if err != nil {
    return fmt.Errorf("step1: %w", err)
}

err = step2()
if err != nil {
    return fmt.Errorf("step2: %w", fmt.Errorf("step1: %w", err))
}

// ✅ 每层只包装一次
```

## 💡 最佳实践

### 1. 错误即数据

```go
// ✅ 把错误当返回值处理
result, err := doSomething()
if err != nil {
    return err
}

// ❌ 不要用 panic 处理普通错误
if somethingWrong {
    panic("something wrong")  // 除非是真正异常
}
```

### 2.  sentinel 错误

```go
// 定义哨兵错误
var ErrNotFound = errors.New("not found")

// 使用
func GetUser(id int) (*User, error) {
    if notFound {
        return nil, ErrNotFound
    }
}

// 检查
if err == ErrNotFound {
    // 特定处理
}
```

### 3. 错误类型

```go
type HTTPError struct {
    Code   int
    Status string
}

func (e HTTPError) Error() string {
    return fmt.Sprintf("HTTP %d: %s", e.Code, e.Status)
}

// 使用
func request() error {
    return HTTPError{Code: 404, Status: "Not Found"}
}

// 检查
var httpErr HTTPError
if errors.As(err, &httpErr) {
    fmt.Println("HTTP 错误码:", httpErr.Code)
}
```

### 4. 资源清理

```go
func process() error {
    file, err := os.Open("file.txt")
    if err != nil {
        return err
    }
    defer file.Close()  // 确保关闭
    
    // 处理文件
}
```

## ✅ 知识点总结

- ✅ 错误是值，不是异常
- ✅ 用 errors.New 或 fmt.Errorf 创建错误
- ✅ 用 %w 包装错误
- ✅ 用 errors.Is 和 errors.As 检查错误
- ✅ 提前返回，避免嵌套
- ✅ 资源清理用 defer
