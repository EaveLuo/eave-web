# 性能优化 - 写出高效的 Go 代码

性能优化需要数据支撑。学会使用工具，找到瓶颈，有的放矢。

## 🔍 性能分析工具

### 1. 基准测试

```go
// benchmark_test.go
func BenchmarkStringConcat(b *testing.B) {
    for i := 0; i < b.N; i++ {
        s := ""
        for j := 0; j < 100; j++ {
            s += "hello"
        }
    }
}

func BenchmarkStringBuilder(b *testing.B) {
    for i := 0; i < b.N; i++ {
        var builder strings.Builder
        for j := 0; j < 100; j++ {
            builder.WriteString("hello")
        }
        _ = builder.String()
    }
}

// 运行
// go test -bench=. -benchmem
```

### 2. CPU Profiling

```go
// main.go
import (
    "runtime/pprof"
    "os"
)

func main() {
    f, _ := os.Create("cpu.prof")
    pprof.StartCPUProfile(f)
    defer pprof.StopCPUProfile()
    
    // 程序逻辑

// 分析
// go tool pprof cpu.prof
```

### 3. 内存 Profiling

```go
import (
    "runtime"
    "runtime/pprof"
)

func memProfile() {
    runtime.GC()
    f, _ := os.Create("mem.prof")
    pprof.WriteHeapProfile(f)
    f.Close()
}
```

## ⚡ 优化技巧

### 1. 字符串拼接

```go
// ❌ 低效
s := ""
for i := 0; i < 1000; i++ {
    s += strconv.Itoa(i)
}

// ✅ 高效
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString(strconv.Itoa(i))
}
s := builder.String()

// 或
parts := make([]string, 1000)
for i := 0; i < 1000; i++ {
    parts[i] = strconv.Itoa(i)
}
s := strings.Join(parts, "")
```

### 2. 切片预分配

```go
// ❌ 频繁扩容
slice := []int{}
for i := 0; i < 10000; i++ {
    slice = append(slice, i)
}

// ✅ 预分配
slice := make([]int, 0, 10000)
for i := 0; i < 10000; i++ {
    slice = append(slice, i)
}
```

### 3. Map 预分配

```go
// ❌ 频繁扩容
m := make(map[string]int)
for i := 0; i < 10000; i++ {
    m[strconv.Itoa(i)] = i
}

// ✅ 预分配
m := make(map[string]int, 10000)
for i := 0; i < 10000; i++ {
    m[strconv.Itoa(i)] = i
}
```

### 4. 避免不必要的转换

```go
// ❌ 重复转换
s := "hello"
for i := 0; i < len(s); i++ {
    b := []byte(s)  // 每次都转换
    b[i] = 'H'
}

// ✅ 一次转换
b := []byte(s)
for i := 0; i < len(b); i++ {
    b[i] = 'H'
}
```

### 5. 使用数组代替切片

```go
// ❌ 切片有额外开销
func process(slice []int) {
    // ...
}

// ✅ 数组（已知大小时）
func processArray(arr [10]int) {
    // ...
}
```

### 6. 减少指针使用

```go
// ❌ 小对象用指针
func process(p *int) {
    // ...
}

// ✅ 小对象用值
func process2(v int) {
    // ...
}
```

### 7. 同步池

```go
var bufferPool = sync.Pool{
    New: func() interface{} {
        return new(bytes.Buffer)
    },
}

func useBuffer() {
    buf := bufferPool.Get().(*bytes.Buffer)
    defer bufferPool.Put(buf)
    
    buf.Reset()
    // 使用 buf
}
```

## 🎯 并发优化

### 1. 限制并发数

```go
// ❌ 无限制
for i := 0; i < 10000; i++ {
    go process(i)
}

// ✅ 限制并发
sem := make(chan struct{}, 100)
for i := 0; i < 10000; i++ {
    sem <- struct{}{}
    go func(id int) {
        defer func() { <-sem }()
        process(id)
    }(i)
}
```

### 2. 批量处理

```go
// ❌ 单个处理
for _, item := range items {
    process(item)
}

// ✅ 批量处理
batchSize := 100
for i := 0; i < len(items); i += batchSize {
    end := i + batchSize
    if end > len(items) {
        end = len(items)
    }
    
    batch := items[i:end]
    processBatch(batch)
}
```

### 3. 无锁设计

```go
// ❌ 锁竞争
var mu sync.Mutex
var count int

func increment() {
    mu.Lock()
    count++
    mu.Unlock()
}

// ✅ 原子操作
var count2 int32

func increment2() {
    atomic.AddInt32(&count2, 1)
}
```

## 📊 内存优化

### 1. 对象复用

```go
// ❌ 频繁分配
func process() {
    data := make([]byte, 1024)
    // 使用
}

// ✅ 复用
var pool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

func process2() {
    data := pool.Get().([]byte)
    defer pool.Put(data)
    // 使用
}
```

### 2. 减少 GC 压力

```go
// ❌ 大量短命对象
for i := 0; i < 1000000; i++ {
    s := fmt.Sprintf("item%d", i)
    process(s)
}

// ✅ 复用缓冲区
var buf []byte
for i := 0; i < 1000000; i++ {
    buf = buf[:0]
    buf = strconv.AppendInt(buf, int64(i), 10)
    process(string(buf))
}
```

```go
// ❌ 不必要的初始化
type Config struct {

c := Config{Name: "", Age: 0}  // 冗余

// ✅ 使用零值
c := Config{}  // 或 var c Config
```

## 💡 最佳实践

### 1. 先正确，后优化

```go
// ✅ 先写出正确的代码
func process(data []int) []int {
    result := make([]int, len(data))
    for i, v := range data {
        result[i] = v * 2
    }
    return result
}

// 然后优化
// 1. 基准测试找到瓶颈
// 2. 针对性优化
// 3. 验证优化效果
```

### 2. 80/20 法则

```
80% 的性能问题来自 20% 的代码
```

**做法：**
1. 用 profiling 找到热点
2. 优化最耗时的 20% 代码
3. 不要过早优化

