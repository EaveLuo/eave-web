# 开发环境搭建 - 从安装到第一个程序

工欲善其事，必先利其器。让我们开始搭建 Go 开发环境！

## 📥 安装 Go

### 1. 下载 Go

访问官网下载对应版本：https://go.dev/dl/

**选择建议：**
- Windows：`.msi` 安装包
- macOS：`.pkg` 安装包 或 `brew install go`
- Linux：`.tar.gz` 压缩包

### 2. 安装步骤

#### Windows

1. 运行下载的 `.msi` 文件
2. 一路点击"Next"
3. 默认安装路径：`C:\Program Files\Go`
4. 安装完成后重启终端

#### macOS

```bash
# 方式一：使用 Homebrew（推荐）
brew install go

# 方式二：下载安装包
# 下载 pkg 文件后双击安装
```

#### Linux

```bash
# 下载（以 1.21 版本为例）
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz

# 解压到 /usr/local
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz

# 添加环境变量
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
```

### 3. 验证安装

```bash
go version
```

输出类似：
```
go version go1.21.0 darwin/amd64
```

## 🛠️ 配置开发环境

### 1. 设置 GOPATH（Go 1.11+ 可选）

Go 1.11+ 引入了 Go Modules，GOPATH 不再是必须的，但了解它仍有价值：

```bash
# 查看当前 GOPATH
go env GOPATH

# 设置 GOPATH（可选）
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
```

### 2. 配置 Go Modules

Go Modules 是 Go 的依赖管理工具（类似 npm、pip）：

```bash
# 启用 Go Modules（Go 1.13+ 默认启用）
export GO111MODULE=on

# 配置国内镜像（加速下载）
go env -w GOPROXY=https://goproxy.cn,direct
```

### 3. 选择编辑器

#### VS Code（推荐）

1. 安装 VS Code
2. 安装 Go 扩展（搜索 "Go"，选择 Microsoft 出品）
3. 扩展会自动安装必要的工具

**优点：**
- 免费、开源
- 强大的智能提示
- 内置调试器
- 丰富的插件生态

#### GoLand

JetBrains 出品的专业 Go IDE：

**优点：**
- 开箱即用
- 强大的重构功能
- 优秀的调试体验
- 数据库工具集成

**缺点：**
- 收费（有试用期）

#### Vim/Neovim

适合终端爱好者：

```vim
' 安装 vim-go 插件
Plug 'fatih/vim-go'
```

## 📝 第一个 Go 程序

### 1. 创建项目目录

```bash
mkdir hello-go
cd hello-go
```

### 2. 初始化 Go Module

```bash
go mod init hello-go
```

生成 `go.mod` 文件：
```go
module hello-go

go 1.21
```

### 3. 编写代码

创建 `main.go`：

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, 世界！")
}
```

### 4. 运行程序

```bash
go run main.go
```

输出：
```
Hello, 世界！
```

### 5. 编译程序

```bash
# 编译成可执行文件
go build main.go

# 运行编译后的程序
./main  # Linux/Mac
main.exe  # Windows
```

## 📁 Go 项目结构

### 基础结构

```
hello-go/
├── go.mod          # 模块定义文件
├── go.sum          # 依赖校验文件
├── main.go         # 主程序
└── pkg/            # 包目录（可选）
    └── utils/
        └── utils.go
```

### 标准项目结构

对于更大的项目，推荐以下结构：

```
my-project/
├── cmd/                    # 可执行程序
│   └── server/
│       └── main.go
├── pkg/                    # 公共包
│   └── database/
│       └── db.go
├── internal/               # 私有包（外部不可导入）
│   └── config/
│       └── config.go
├── go.mod
└── go.sum
```

**目录说明：**
- `cmd/`：存放可执行程序的入口
- `pkg/`：存放可被外部使用的包
- `internal/`：存放仅项目内部使用的包

## 🔧 常用 Go 命令

### 基础命令

```bash
# 运行程序
go run main.go

# 编译程序
go build main.go

# 安装程序到 GOPATH/bin
go install main.go

# 格式化代码
go fmt main.go

# 格式化整个项目
go fmt ./...
```

### 依赖管理

```bash
# 初始化模块
go mod init <module-name>

# 下载依赖
go mod download

# 整理依赖（删除未使用的）
go mod tidy

# 查看依赖
go list -m all

# 升级依赖
go get -u <package>
```

### 测试和检查

```bash
# 运行测试
go test

# 运行测试（详细输出）
go test -v

# 代码检查
go vet ./...

# 查看代码覆盖率
go test -cover
```

## 🐛 常见问题

### 1. 下载依赖太慢

**解决方案：** 使用国内镜像

```bash
go env -w GOPROXY=https://goproxy.cn,direct
```

### 2. 命令找不到

**问题：** `go: command not found`

**解决方案：** 添加环境变量

```bash
# Linux/Mac
export PATH=$PATH:/usr/local/go/bin

# Windows
# 在系统环境变量中添加 C:\Program Files\Go\bin
```

### 3. 版本不兼容

**问题：** 依赖需要更高版本的 Go

**解决方案：** 升级 Go

```bash
# macOS
brew upgrade go

# Linux
# 重新下载新版本并替换
```
