# 并发最佳实践 - 避免常见陷阱

## ⚠️ 常见并发陷阱

### 1. Goroutine 泄漏

```go
// ❌ 错误：Goroutine 泄漏
func worker(ch <-chan int) {
    for v := range ch {
        process(v)
    }
}

func main() {
    ch := make(chan int)
    go worker(ch)
    ch <- 1
    // 忘记关闭 channel，worker 永远阻塞
}

// ✅ 正确
close(ch)  // 关闭 channel，worker 退出
```

### 2. 竞态条件

```go
// ❌ 错误：竞态条件
var counter int
go func() { counter++ }()
go func() { counter++ }()

// ✅ 正确：使用 Mutex
var mu sync.Mutex
var counter int
mu.Lock()
counter++
mu.Unlock()

// ✅ 正确：使用 atomic
var counter int64
atomic.AddInt64(&counter, 1)
```

### 3. 死锁

```go
// ❌ 错误：死锁
ch1 := make(chan int)
ch2 := make(chan int)
go func() { ch1 <- 1; <-ch2 }()
go func() { ch2 <- 2; <-ch1 }()

// ✅ 正确：使用 select
select {
case ch1 <- 1:
    <-ch2
case <-time.After(time.Second):
    // 超时处理
}
```

## 🎯 并发模式

### Worker Pool

```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for job := range jobs {
        results <- job * 2
    }
}

// 启动工作池
for w := 1; w <= 3; w++ {
    go worker(w, jobs, results)
}
```
