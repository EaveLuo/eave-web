# 接口 - Go 的灵魂所在

接口是 Go 最强大的特性之一。它定义了行为的规范，而不关心具体实现。

## 📐 接口定义

```go
// 定义接口
type Speaker interface {
    Speak() string
}

// 实现接口（无需声明！）
type Person struct {
    Name string
}

func (p Person) Speak() string {
    return "我是" + p.Name
}

// Person 自动实现了 Speaker 接口
```

## 🎯 隐式实现

Go 的接口是**隐式**实现的：

```go
type Writer interface {
    Write([]byte) (int, error)
}

type File struct{}

func (f File) Write(data []byte) (int, error) {
    // 实现写入逻辑
    return len(data), nil
}

// File 自动实现了 Writer，无需声明！
// 这就是"鸭子类型"：像鸭子一样走路、游泳、叫，就是鸭子
```

## 🔀 空接口

`interface{}` 可以接受任何类型：

```go
// 接受任何类型
func PrintAny(v interface{}) {
    fmt.Println(v)
}

PrintAny(42)        // int
PrintAny("hello")   // string
PrintAny([]int{1,2,3})  // slice
```

**实际应用：**

```go
// JSON 解析
var data interface{}
json.Unmarshal(jsonData, &data)

// 通用容器
type Stack struct {
    items []interface{}
}

func (s *Stack) Push(v interface{}) {
    s.items = append(s.items, v)
}

func (s *Stack) Pop() interface{} {
    // ...
}
```

## 🔍 类型断言

从空接口获取具体类型：

```go
var v interface{} = "hello"

// 方式 1：断言（可能 panic）
s := v.(string)

// 方式 2：安全断言（推荐）
s, ok := v.(string)
if ok {
    fmt.Println("是字符串:", s)
}

// 方式 3：type switch
switch t := v.(type) {
case string:
    fmt.Println("字符串:", t)
case int:
    fmt.Println("整数:", t)
default:
    fmt.Println("未知类型")
}
```

## 🎨 接口组合

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

// 组合接口
type ReadWriter interface {
    Reader
    Writer
}
```

## 💡 常用接口

### 1. fmt.Stringer

```go
type Person struct {
    Name string
    Age  int
}

func (p Person) String() string {
    return fmt.Sprintf("%s(%d)", p.Name, p.Age)
}

// 自动被 fmt 调用
fmt.Println(Person{"张三", 25})  // 张三 (25)
```

### 2. error 接口

```go
type error interface {
    Error() string
}

// 自定义错误
type MyError struct {
    Code    int
    Message string
}

func (e MyError) Error() string {
    return fmt.Sprintf("[%d] %s", e.Code, e.Message)
}

// 使用
func doSomething() error {
    return MyError{Code: 404, Message: "Not Found"}
}
```

### 3. sort.Interface

```go
type PersonList []Person

func (p PersonList) Len() int           { return len(p) }
func (p PersonList) Swap(i, j int)      { p[i], p[j] = p[j], p[i] }
func (p PersonList) Less(i, j int) bool { return p[i].Age < p[j].Age }

// 排序
sort.Sort(PersonList{p1, p2, p3})
```

## 🐛 常见错误

### 1. 接口值比较

```go
var a interface{} = []int{1, 2}
var b interface{} = []int{1, 2}

// ❌ panic：切片不能比较
// fmt.Println(a == b)

// ✅ 正确
fmt.Println(reflect.DeepEqual(a, b))
```

### 2. nil 接口值

```go
var w Writer  // nil 接口

// ❌ 可能 panic
if w == nil {  // true
    w.Write([]byte("hello"))  // panic!
}

// ✅ 检查内部值
type Writer interface {
    Write([]byte) (int, error)
}

type File struct{}
func (f *File) Write(data []byte) (int, error) { return len(data), nil }

var w2 Writer = (*File)(nil)  // 非 nil 接口，nil 值
fmt.Println(w2 == nil)  // false!
```

### 3. 过度使用空接口

```go
// ❌ 不好：丢失类型安全
func Process(data interface{}) {
    // ...
}

// ✅ 推荐：使用泛型或具体类型
func ProcessString(data string) {
    // ...
}

// Go 1.18+ 泛型
func Process[T any](data T) {
    // ...
}
```

## 💡 最佳实践

### 1. 小接口原则

```go
// ✅ 推荐：小接口
type Reader interface {
    Read(p []byte) (n int, err error)
}

// ❌ 避免：大接口
type DataProcessor interface {
    Read() error
    Write() error
    Process() error
    Validate() error
    // ... 太多方法
}
```

### 2. 接口在消费侧定义

```go
// ✅ 推荐：接口在需要使用的一方定义
package payment

type Payer interface {
    Pay(amount int) error
}

func ProcessPayment(p Payer) {
    p.Pay(100)
}

// 实现可以在其他地方
package alipay

type Alipay struct{}
func (a Alipay) Pay(amount int) error { ... }
```

### 3. 不要过早抽象

```go
// ❌ 过度设计
type Database interface {
    Connect() error
    Query(sql string) ([]Row, error)
    Close() error
}

// ✅ 简单直接
type DB struct {
    // 具体实现
}

func (db *DB) Query(sql string) ([]Row, error) {
    // ...
}
```
