# 项目结构规范

本文档描述 eave-web 项目的目录结构和组织规范。

│   └── website/                # 博客更新日志

│   ├── front-end/              # 前端文档
│   ├── back-end/               # 后端文档
│   └── operation/              # 运维文档
├── i18n/                       # 国际化配置
│   ├── en/

## 📂 核心目录说明

### src/components/

存放自定义 React 组件。

**组织原则：**
- 按功能模块分组
- 每个组件独立文件夹
- 包含组件实现和样式

```
components/
├── Button/
│   ├── index.tsx           # 组件实现
│   └── styles.module.css   # 组件样式
├── Layout/
│   ├── Header.tsx
│   ├── Footer.tsx
│   └── index.ts
└── index.ts                # 统一导出
```

### src/theme/

覆盖 Docusaurus 默认主题组件。

**常见覆盖：**
- `Navbar` - 导航栏
- `Footer` - 页脚
- `DocItem` - 文档项
- `BlogPostPage` - 博客文章页

```tsx
// src/theme/Navbar/index.tsx
import OriginalNavbar from '@theme-original/Navbar';

export default function Navbar(props) {
  return <OriginalNavbar {...props} />;
}
```

### docs/

技术文档目录，按领域分类。

```
docs/
├── front-end/              # 前端技术
│   ├── intro.md
│   ├── react/              # React 相关
│   ├── docusaurus/         # Docusaurus 相关
│   └── typescript/         # TypeScript 相关
├── back-end/               # 后端技术
│   ├── intro.md
│   └── node/               # Node.js 相关
└── operation/              # 运维相关
    ├── intro.md
    ├── linux/              # Linux 相关
    └── nginx/              # Nginx 相关
```

### blog/

博客文章目录。

**命名规范：**
- `YYYY-MM-DD-title.md`
- 或版本更新：`0.1.0.md`, `0.2.0.md`

**Front Matter：**
```md
---
title: 文章标题
description: 文章描述
authors:
  - name: Eave Luo
    url: https://github.com/EaveLuo
tags: [react, typescript]
---
```

## 🔧 配置文件

### docusaurus.config.ts

主配置文件，包含：
- 站点元数据（title, url, baseUrl）
- 主题配置（navbar, footer, prism）
- 插件配置
- 预设配置（docs, blog, theme）

### sidebars.ts

侧边栏配置，定义文档导航结构。

```ts
const sidebars: SidebarsConfig = {
  frontEndSidebar: [{ type: 'autogenerated', dirName: 'front-end' }],
  backEndSidebar: [{ type: 'autogenerated', dirName: 'back-end' }],
  operationSidebar: [{ type: 'autogenerated', dirName: 'operation' }],
};
```

### tailwind.config.js

TailwindCSS 配置。

```js
module.exports = {
  content: ['./src/**/*.{js,jsx,ts,tsx}'],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

### tsconfig.json

TypeScript 配置。

```json
{
  "extends": "@docusaurus/tsconfig",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@site/*": ["./*"]
    }
  }
}
```

## 📝 编码规范

### 文件命名

- 组件：`PascalCase.tsx` (e.g., `UserProfile.tsx`)
- 工具函数：`camelCase.ts` (e.g., `utils.ts`)
- 样式：`*.module.css` 或 `*.css`
- 配置：`*.config.js/ts`

### 导入顺序

```tsx
// 1. React
import React from 'react';

// 2. Docusaurus
import useDocusaurusContext from '@docusaurus/useDocusaurusContext';

// 3. 第三方库
import clsx from 'clsx';

// 4. 内部组件
import Button from '@site/src/components/Button';

// 5. 样式
import styles from './styles.module.css';
```

### 导出规范

```tsx
// ✅ 默认导出组件
export default function Component() {}

// ✅ 命名导出类型
export type { ComponentProps } from './Component';

// ✅ 统一导出（index.ts）
export { default as Component } from './Component';
export type { ComponentProps } from './Component';
```

## 🚀 开发流程

### 1. 本地开发

```bash
yarn install
yarn start
```

### 2. 构建

```bash
yarn build
```

### 3. 类型检查

```bash
yarn typecheck
```

### 4. 部署

Push 到 master 分支自动触发 GitHub Actions 部署到 Vercel。
