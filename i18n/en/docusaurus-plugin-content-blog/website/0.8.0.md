```typescript
function extractDescription(frontMatterDesc, content, maxLength = 120) {
  // Priority 1: front matter description
  if (frontMatterDesc?.trim()) {
    return truncate(frontMatterDesc, maxLength);
  }
  
  // Priority 2: extract from content (remove markdown)
  const text = content
    .replace(/```[\s\S]*?```/g, ' ')     // code blocks
    .replace(/`[^`]*`/g, ' ')             // inline code
    .replace(/!\[[^\]]*\]\([^)]+\)/g, ' ') // images
    .replace(/\[([^\]]+)\]\([^)]+\)/g, '$1') // links
    // ... more cleanup rules
  
  return truncate(text, maxLength);
}
```

### ⚡ Animation Performance Optimization

#### Core Issues Diagnosed

Performance analysis revealed the following bottlenecks on the homepage:

| Component | Issue | Impact |
|-----------|-------|--------|
| Particles | 100 particles + per-frame redraw + high-frequency mouse events | High GPU usage, lag on low-end devices |
| TypingAnimation | setInterval 16ms forced reflow | Main thread blocking |
| ArticleCard | 12 cards triggering Framer Motion simultaneously | Excessive compositing layers, high memory usage |
| Name mouse tracking | setProperty on every mousemove | Forced style recalculation |

#### Optimization Strategy & Implementation

**1. Particles Effect**

```typescript
// Before
quantity: 100        // particle count
animate: setInterval // timer-driven

// After
quantity: 30         // 70% reduction
animate: requestAnimationFrame // frame-synced

// Additional optimizations
const isActive = useRef(true);           // lifecycle control
const dpr = Math.min(window.devicePixelRatio, 2); // limit DPR
useCallback + useMemo                    // memoized calculations
RAF-throttled mouse events               // reduced event frequency
```

**2. TypingAnimation**

```typescript
// Before
useEffect(() => {
  const interval = setInterval(() => {
    setDisplayedText(prev => prev + text.charAt(i));
    setI(i + 1);
  }, duration);
}, [i]);

// After
const animate = useCallback((currentTime: number) => {
  const deltaTime = currentTime - lastTimeRef.current;
  if (deltaTime >= duration) {
    setDisplayedText(text.slice(0, indexRef.current + 1));
    indexRef.current += 1;
    lastTimeRef.current = currentTime;
  }
  rafRef.current = requestAnimationFrame(animate);
}, []);
```

**3. ArticleCard Entrance Animation**

```typescript
// Before - Framer Motion whileInView
<motion.article
  initial={{ opacity: 0, y: 20 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true }}
  transition={{ duration: 0.4, delay: index * 0.1 }}
/>

// After - CSS animation + useInView
const isInView = useInView(cardRef, { once: true, margin: "-50px" });

// CSS
.card {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.4s ease, transform 0.4s ease;
}
.cardVisible {
  opacity: 1;
  transform: translateY(0);
}
```

**4. Hero Mouse Lighting Effect**

```typescript
// Before
onMouseMove={(e) => {
  e.currentTarget.style.setProperty('--x', `${e.clientX}px`);
  e.currentTarget.style.setProperty('--y', `${e.clientY}px`);
}}

// After
const handleMouseMove = useCallback((e: React.MouseEvent) => {
  if (rafRef.current) return;  // RAF throttle
  rafRef.current = requestAnimationFrame(() => {
    setMousePos({ x: e.clientX - bounding.x, y: e.clientY - bounding.y });
    rafRef.current = 0;
  });
}, []);
```

#### Key Code Snippets

**Lazy-loaded Particle Effect:**

```typescript
// Using IntersectionObserver + requestIdleCallback
useEffect(() => {
  const observer = new IntersectionObserver((entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        requestIdleCallback(() => setShowParticles(true), { timeout: 2000 });
        observer.disconnect();
      }
    });
  });
  
  // 3-second fallback
  const fallback = setTimeout(() => setShowParticles(true), 3000);
}, []);
```

### 🎉 Summary

Version 0.8.0 is an upgrade that balances **experience and performance**. The homepage card modules make content display more intuitive, documentation standardization ensures seamless switching for multilingual users, and deep performance optimization makes the overall experience silkier and smoother.

Notably, all optimizations follow **progressive enhancement** principles:
- Support `prefers-reduced-motion` media query, respecting user preferences
- Use `requestIdleCallback` to delay non-critical resource loading
- Keep original components as fallback for easy rollback

Performance optimization has no end, only continuous iteration 🚀
