# Development Environment Setup - From Installation to First Program

To do a good job, one must first sharpen one's tools. Let's set up the Go development environment!

## InboxTray Installing Go

### 1. Download Go

Visit the official website to download the appropriate version: https://go.dev/dl/

**Selection Guide:**
- Windows: `.msi` installer
- macOS: `.pkg` installer or `brew install go`
- Linux: `.tar.gz` archive

### 2. Installation Steps

#### Windows

1. Run the downloaded `.msi` file
2. Click "Next" all the way through
3. Default installation path: `C:\Program Files\Go`
4. Restart terminal after installation

#### macOS

```bash
# Method 1: Using Homebrew (recommended)
brew install go

# Method 2: Download installer
# Double-click the pkg file after downloading
```

#### Linux

```bash
# Download (using version 1.21 as example)
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz

# Extract to /usr/local
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz

# Add to PATH
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc
```

### 3. Verify Installation

```bash
go version
```

Output should look like:
```
go version go1.21.0 darwin/amd64
```

## HammerAndWrench Configure Development Environment

### 1. Set GOPATH (Optional for Go 1.11+)

Go 1.11+ introduced Go Modules, making GOPATH optional, but it's still worth understanding:

```bash
# Check current GOPATH
go env GOPATH

# Set GOPATH (optional)
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
```

### 2. Configure Go Modules

Go Modules is Go's dependency management tool (similar to npm, pip):

```bash
# Enable Go Modules (default in Go 1.13+)
export GO111MODULE=on

# Configure mirror for faster downloads (China)
go env -w GOPROXY=https://goproxy.cn,direct
```

### 3. Choose an Editor

#### VS Code (Recommended)

1. Install VS Code
2. Install Go extension (search "Go", select Microsoft version)
3. Extension will auto-install necessary tools

**Pros:**
- Free and lightweight
- Rich plugin ecosystem
- Excellent Go support
- Built-in debugging

#### GoLand (JetBrains)

Professional IDE specifically for Go:

**Pros:**
- Powerful refactoring
- Advanced debugging
- Database tools built-in
- Commercial support

**Cons:**
- Paid software (except for students/open source)
- Heavier resource usage

#### Vim/Neovim

For terminal enthusiasts:

```bash
# Install vim-go plugin
# Using vim-plug:
Plug 'fatih/vim-go', { 'do': ':GoUpdateBinaries' }
```

## Memo First Go Program

### 1. Create Project Directory

```bash
mkdir hello-go
cd hello-go
```

### 2. Initialize Go Module

```bash
go mod init hello-go
```

This creates `go.mod` file:
```go
module hello-go

go 1.21
```

### 3. Write Code

Create `main.go`:

```go
package main

import "fmt"

func main() {
    fmt.Println("Hello, World!")
}
```

### 4. Run Program

```bash
go run main.go
```

Output:
```
Hello, World!
```

### 5. Compile Program

```bash
# Compile into executable
go build main.go

# Run compiled program
./main  # Linux/Mac
main.exe  # Windows
```

## CardIndexDividers Go Project Structure

### Basic Structure

```
hello-go/
├── go.mod          # Module definition file
├── go.sum          # Dependency checksum file
├── main.go         # Main program
└── pkg/            # Package directory (optional)
    └── utils/
        └── utils.go
```

### Standard Project Structure

For larger projects, recommend the following structure:

```
my-project/
├── cmd/                    # Executable programs
│   └── server/
│       └── main.go
├── pkg/                    # Public packages
│   └── database/
│       └── db.go
├── internal/               # Private packages (not importable externally)
│   └── config/
│       └── config.go
├── go.mod
└── go.sum
```

**Directory Descriptions:**
- `cmd/`: Store entry points for executables
- `pkg/`: Store packages usable by external projects
- `internal/`: Store packages for internal use only

## Wrench Common Go Commands

### Basic Commands

```bash
# Run program
go run main.go

# Compile program
go build main.go

# Install program to GOPATH/bin
go install main.go

# Format code
go fmt main.go

# Format entire project
go fmt ./...
```

### Dependency Management

```bash
# Initialize module
go mod init <module-name>

# Download dependencies
go mod download

# Tidy dependencies (remove unused)
go mod tidy

# View dependencies
go list -m all

# Upgrade dependencies
go get -u <package>
```

### Testing and Checking

```bash
# Run tests
go test

# Run tests (verbose output)
go test -v

# Code checking
go vet ./...

# View code coverage
go test -cover
```

## Bug Common Issues

### 1. Slow Dependency Downloads

**Solution:** Use domestic mirror

```bash
go env -w GOPROXY=https://goproxy.cn,direct
```

### 2. Command Not Found

**Problem:** `go: command not found`

**Solution:** Add environment variable

```bash
# Linux/Mac
export PATH=$PATH:/usr/local/go/bin

# Windows
# Add C:\Program Files\Go\bin to system environment variables
```

### 3. Version Incompatibility

**Problem:** Dependencies require higher Go version

**Solution:** Upgrade Go

```bash
# macOS
brew upgrade go

# Linux
# Re-download new version and replace
```
