# Reflection - Runtime Type Introspection

Reflection allows programs to inspect and manipulate types at runtime. This is an advanced Go feature, powerful but should be used with caution.

## MagnifyingGlassTiltedLeft What is Reflection?

Reflection allows programs at runtime to:
- Inspect variable types
- Access struct fields
- Call methods
- Modify values

```go
import "reflect"

var x float64 = 3.4
v := reflect.ValueOf(x)

fmt.Println("Type:", v.Type())  // float64
fmt.Println("Kind:", v.Kind())  // float64
fmt.Println("Value:", v.Float())   // 3.4
```

## Target Core API

### 1. reflect.TypeOf

```go
type Person struct {
    Name string
    Age  int
}

p := Person{Name: "John", Age: 25}

t := reflect.TypeOf(p)
fmt.Println("Type:", t)        // main.Person
fmt.Println("Kind:", t.Kind()) // struct

// Iterate over fields
for i := 0; i < t.NumField(); i++ {
    field := t.Field(i)
    fmt.Printf("Field: %s, Type: %s\n", field.Name, field.Type)
}
```

### 2. reflect.ValueOf

```go
p := Person{Name: "John", Age: 25}
v := reflect.ValueOf(p)

// Access fields
name := v.Field(0).String()
age := v.Field(1).Int()

fmt.Println(name, age)  // John 25
```

### 3. Modifying Values

```go
p := Person{Name: "John", Age: 25}
v := reflect.ValueOf(&p).Elem()  // Need pointer to modify

v.Field(0).SetString("Jane")
v.Field(1).SetInt(26)

fmt.Println(p)  // {Jane 26}
```

## ArtistPalette Practical Applications

### 1. Generic JSON Parsing

```go
func parseJSON(data []byte, v interface{}) error {
    rv := reflect.ValueOf(v)
    
    // Check if it's a pointer
    if rv.Kind() != reflect.Ptr {
        return errors.New("must pass a pointer")
    }
    
    // Parse logic...
    return nil
}
```

### 2. Struct Validation

```go
func validate(v interface{}) error {
    val := reflect.ValueOf(v)
    typ := val.Type()
    
    for i := 0; i < val.NumField(); i++ {
        field := val.Field(i)
        tag := typ.Field(i).Tag.Get("validate")
        
        if tag == "required" && field.IsZero() {
            return fmt.Errorf("%s cannot be empty", typ.Field(i).Name)
        }
    }
    
    return nil
}

// Usage
type User struct {
    Name string `validate:"required"`
    Age  int    `validate:"required"`
}

err := validate(User{})  // Error: Name cannot be empty
```

### 3. Generic Map Conversion

```go
func structToMap(v interface{}) map[string]interface{} {
    result := make(map[string]interface{})
    val := reflect.ValueOf(v)
    typ := val.Type()
    
    for i := 0; i < val.NumField(); i++ {
        field := typ.Field(i)
        value := val.Field(i).Interface()
        result[field.Name] = value
    }
    
    return result
}
```

### 4. Dynamic Method Invocation

```go
type Calculator struct{}

func (c Calculator) Add(a, b int) int {
    return a + b
}

func callMethod(obj interface{}, methodName string, args ...interface{}) []reflect.Value {
    v := reflect.ValueOf(obj)
    method := v.MethodByName(methodName)
    
    if !method.IsValid() {
        panic("method does not exist")
    }
    
    in := make([]reflect.Value, len(args))
    for i, arg := range args {
        in[i] = reflect.ValueOf(arg)
    }
    
    return method.Call(in)
}

// Usage
calc := Calculator{}
results := callMethod(calc, "Add", 10, 20)
fmt.Println(results[0].Int())  // 30
```

## Bug Common Mistakes

### 1. Unmodifiable Values

```go
x := 42
v := reflect.ValueOf(x)

// CrossMark panic
v.SetInt(100)  // Cannot modify

// WhiteCheckMark Correct
v2 := reflect.ValueOf(&x).Elem()
v2.SetInt(100)
```

### 2. Type Assertion Failure

```go
var x interface{} = 42
v := reflect.ValueOf(x)

// CrossMark Wrong
// floatVal := v.Float()  // panic

// WhiteCheckMark Correct
if v.Kind() == reflect.Int {
    intVal := v.Int()
}
```

### 3. Performance Issues

```go
// CrossMark Reflection is slow
for i := 0; i < 1000000; i++ {
    v := reflect.ValueOf(obj)
    v.Field(0).Int()
}

// WhiteCheckMark Direct access
for i := 0; i < 1000000; i++ {
    _ = obj.Field0
}
```

## LightBulb Best Practices

### 1. Prefer Type Assertions

```go
// CrossMark Overuse of reflection
func getName(v interface{}) string {
    return reflect.ValueOf(v).FieldByName("Name").String()
}

// WhiteCheckMark Type assertion
func getName2(v interface{}) string {
    if p, ok := v.(Person); ok {
        return p.Name
    }
    return ""
}
```

### 2. Cache Reflection Results

```go
var fieldCache = make(map[reflect.Type][]reflect.StructField)

func getFields(t reflect.Type) []reflect.StructField {
    if fields, ok := fieldCache[t]; ok {
        return fields
    }
    
    fields := make([]reflect.StructField, t.NumField())
    for i := 0; i < t.NumField(); i++ {
        fields[i] = t.Field(i)
    }
    fieldCache[t] = fields
    return fields
}
```

### 3. Clear Error Handling

```go
func safeReflect(v interface{}) (err error) {
    defer func() {
        if r := recover(); r != nil {
            err = fmt.Errorf("reflection error: %v", r)
        }
    }()
    
    // Reflection operations
    reflect.ValueOf(v).Elem().Field(0).String()
    
    return nil
}
```

## WhiteCheckMark Key Points Summary

- WhiteCheckMark TypeOf gets type information
- WhiteCheckMark ValueOf gets values
- WhiteCheckMark Need pointer to modify values
- WhiteCheckMark Reflection has lower performance, use cautiously
- WhiteCheckMark Prefer type assertions
- WhiteCheckMark Cache reflection results for better performance
