# Program Structure - Packages, Imports, and Visibility

## 📦 What is a Package?

Packages are the basic organization units in Go programs.

### Package Declaration

```go
package main  // Executable program

package fmt   // Library package
```

**Rules:**
- `package main`: Executable, must have `main()` function
- Other names: Library packages for reuse

## 👁️ Visibility Rules (Export)

Go uses **capitalization** for visibility control:

```go
package utils

// ✅ Exported (starts with uppercase)
var Config string      // Accessible outside
func Helper() {}       // Callable outside
type Person struct {}  // Usable outside

// ❌ Unexported (starts with lowercase)
var config string      // Package-private
func helper() {}       // Package-private
type person struct {}  // Package-private
```

## 🔗 Importing Packages

```go
import (
    "fmt"                           // Standard library
    "github.com/gin-gonic/gin"      // Third-party
    myfmt "myproject/utils"         // Alias
    _ "database/sql"                // Anonymous import
)
```

**⚠️ Rule**: Imported packages must be used!

## 🔄 Package Initialization

```go
// init() runs automatically
func init() {
    fmt.Println("Initializing...")
}
```

**Order:**
1. Initialize imported packages
2. Initialize package-level variables
3. Run init() functions
