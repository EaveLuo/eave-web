# Methods - Giving Types Behavior

Methods are functions with receivers. Go's methods give types behavior, achieving object-oriented effects.

## Package Method Definition

```go
type Person struct {
    Name string
    Age  int
}

// Method definition (receiver comes before function name)
func (p Person) Greet() {
    fmt.Printf("Hello, I'm %s!\n", p.Name)
}

// Call
p := Person{Name: "John", Age: 25}
p.Greet()  // Hello, I'm John!
```

## ArrowsClockwise Value Receiver vs Pointer Receiver

### Value Receiver

```go
func (p Person) GetAge() int {
    return p.Age  // Read value
}

func (p Person) SetAge(age int) {
    p.Age = age  // CrossMark Only modifies copy, original unchanged
}
```

### Pointer Receiver

```go
func (p *Person) GetAge2() int {
    return p.Age
}

func (p *Person) SetAge2(age int) {
    p.Age = age  // WhiteCheckMark Modifies original object
}
```

### Selection Rules

```go
// Use pointer receiver:
- Need to modify the receiver
- Receiver is large struct (avoid copying)
- Maintain consistency (all methods of same type)

// Use value receiver:
- Don't need to modify the receiver
- Receiver is small type (int, string)
- Receiver is slice, Map, Channel
```

## Target Method Sets

```go
type Counter struct {
    count int
}

func (c Counter) Get() int        { return c.count }
func (c *Counter) Inc()           { c.count++ }
func (c *Counter) Reset()         { c.count = 0 }

// Counter's method set: Get
// *Counter's method set: Get, Inc, Reset

c := Counter{}
c.Get()     // WhiteCheckMark
c.Inc()     // WhiteCheckMark Go automatically takes address

pc := &Counter{}
pc.Get()    // WhiteCheckMark
pc.Inc()    // WhiteCheckMark
```

## ArtistPalette Encapsulation

Go uses capitalization to control visibility:

```go
type Account struct {
    balance int  // Private field
}

// Private method
func (a *Account) validate(amount int) bool {
    return amount > 0
}

// Public method
func (a *Account) Deposit(amount int) error {
    if !a.validate(amount) {
        return errors.New("invalid amount")
    }
    a.balance += amount
    return nil
}

func (a *Account) GetBalance() int {
    return a.balance
}
```

## LightBulb Practical Examples

### 1. Method Chaining

```go
type Builder struct {
    result string
}

func (b *Builder) Append(s string) *Builder {
    b.result += s
    return b  // Return self
}

func (b *Builder) Build() string {
    return b.result
}

// Chain calls
result := (&Builder{}).Append("Hello").Append(" ").Append("World").Build()
```

### 2. Implementing Interfaces

```go
type Stringer interface {
    String() string
}

type Person struct {
    Name string
    Age  int
}

func (p Person) String() string {
    return fmt.Sprintf("%s(%d)", p.Name, p.Age)
}

// Person implements Stringer interface
```

### 3. Methods as Values

```go
type Handler func(string)

type Server struct{}

func (s *Server) HandleRequest(req string) {
    fmt.Println("Processing request:", req)
}

func (s *Server) GetHandler() Handler {
    return s.HandleRequest
}

// Usage
server := &Server{}
handler := server.GetHandler()
handler("GET /api/users")
```

## Bug Common Mistakes

### 1. Receiver Type Mismatch

```go
type Counter struct{ count int }

func (c Counter) Inc() { c.count++ }

c := Counter{}
c.Inc()  // WhiteCheckMark But c.count unchanged!

// Should use pointer receiver
func (c *Counter) Inc2() { c.count++ }
```

### 2. Mixing Receiver Types

```go
// CrossMark Inconsistent
func (c Counter) Get() int { ... }
func (c *Counter) Set(v int) { ... }

// WhiteCheckMark Consistent
func (c *Counter) Get() int { ... }
func (c *Counter) Set(v int) { ... }
```

### 3. Modifying Slices in Methods

```go
type Data struct {
    nums []int
}

// WhiteCheckMark Slice is reference, can modify directly
func (d *Data) Append(n int) {
    d.nums = append(d.nums, n)
}
```

## WhiteCheckMark Key Points Summary

- WhiteCheckMark Method = Function + Receiver
- WhiteCheckMark Value receiver doesn't modify original object
- WhiteCheckMark Pointer receiver modifies original object
- WhiteCheckMark Capitalization controls visibility
- WhiteCheckMark Methods implement interfaces
- WhiteCheckMark Maintain consistency
