# Methods - Giving Types Behavior

Methods are functions with a receiver. Go's methods give types behavior, achieving object-oriented effects.

## 📦 Method Definition

```go
type Person struct {
    Name string
    Age  int
}

// Method definition (receiver before function name)
func (p Person) Greet() {
    fmt.Printf("Hello, I'm %s!\n", p.Name)
}

// Call
p := Person{Name: "John", Age: 25}
p.Greet()  // Hello, I'm John!
```

## 🔀 Value Receiver vs Pointer Receiver

### Value Receiver

```go
func (p Person) GetAge() int {
    return p.Age  // Read value
}

func (p Person) SetAge(age int) {
    p.Age = age  // ❌ Only modifies copy, original unchanged
}
```

### Pointer Receiver

```go
func (p *Person) GetAge2() int {
    return p.Age
}

func (p *Person) SetAge2(age int) {
    p.Age = age  // ✅ Modifies original
}
```

### Selection Rules

Use pointer receivers when:
- Need to modify the receiver
- Receiver is large struct (avoid copying)
- Consistency (all methods of same type)

Use value receivers when:
- Small, immutable types
- Don't need to modify
- Thread safety concerns

## 🎯 Method Examples

```go
type Rectangle struct {
    Width  float64
    Height float64
}

// Value receiver - doesn't modify
func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

// Pointer receiver - modifies
func (r *Rectangle) Scale(factor float64) {
    r.Width *= factor
    r.Height *= factor
}

// Usage
rect := Rectangle{Width: 10, Height: 5}
fmt.Println(rect.Area())     // 50
rect.Scale(2)
fmt.Println(rect.Area())     // 200
```

## 🔗 Method on Non-Struct Types

Go allows methods on any named type:

```go
// Define new type based on string
type MyString string

// Method on MyString
func (s MyString) Upper() string {
    return strings.ToUpper(string(s))
}

// Usage
str := MyString("hello")
fmt.Println(str.Upper())  // "HELLO"
```

## 💡 Best Practices

### 1. Be Consistent

```go
// ✅ Good: All methods use same receiver type
type User struct {
    Name string
}

func (u *User) SetName(name string)  // pointer
func (u *User) GetName() string      // pointer
func (u *User) Validate() error      // pointer

// ❌ Bad: Mixed receivers
type User struct { }
func (u User) GetName() string       // value
func (u *User) SetName(name string)  // pointer
```

### 2. Use Pointer for Large Structs

```go
// Large struct
type Image struct {
    Pixels [][]Color
    Width  int
    Height int
}

// ✅ Good: Avoid copying large data
func (img *Image) Resize(w, h int)
func (img *Image) Filter(filter Filter)

// ❌ Bad: Copies entire image each call
func (img Image) Resize(w, h int)
```

### 3. Document Mutability

```go
// Save persists the user to database.
// This method modifies the user's LastSaved field.
func (u *User) Save() error

// Clone returns a deep copy of the user.
// This method does not modify the original.
func (u User) Clone() User
```

## 📊 Comparison with Classes

| Feature | Class (OOP) | Go Method |
|---------|-------------|-----------|
| Definition | Inside class | Separate from type |
| Receiver | `this` implicit | Explicit `(t Type)` |
| Inheritance | Supported | Composition only |
| Overriding | Supported | No inheritance |

**Go philosophy:** Methods are just functions with a receiver. Keep it simple.
