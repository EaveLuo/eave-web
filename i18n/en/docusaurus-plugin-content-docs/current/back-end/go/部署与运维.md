# Deployment and Operations - From Development to Production

## 🔨 Cross-Compilation

```bash
# Linux AMD64
GOOS=linux GOARCH=amd64 go build -o app-linux

# macOS ARM64
GOOS=darwin GOARCH=arm64 go build -o app-macos

# Windows AMD64
GOOS=windows GOARCH=amd64 go build -o app.exe

# Optimize binary size
go build -ldflags="-s -w" -o app
```

## 🐳 Docker Containerization

```dockerfile
# Build stage
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 go build -o myapp

# Runtime stage
FROM alpine:latest
COPY --from=builder /app/myapp /
CMD ["/myapp"]
```

## 📊 Health Checks

```go
func healthHandler(w http.ResponseWriter, r *http.Request) {
    status := map[string]string{
        "status": "healthy",
        "time": time.Now().Format(time.RFC3339),
    }
    json.NewEncoder(w).Encode(status)
}

http.Handle("/health", http.HandlerFunc(healthHandler))
```

## 📈 Monitoring

### Prometheus Metrics

```go
import "github.com/prometheus/client_golang/prometheus"

var requestsTotal = prometheus.NewCounter(
    prometheus.CounterOpts{
        Name: "http_requests_total",
        Help: "Total HTTP requests",
    },
)

http.Handle("/metrics", promhttp.Handler())
```

## 🔄 CI/CD Pipeline

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - uses: actions/setup-go@v4
    - run: go test ./...
    - run: go build
```

## 🎯 Best Practices

1. **Graceful shutdown** - Handle SIGTERM properly
2. **Structured logging** - Use JSON format
3. **Configuration management** - Environment variables
4. **Resource limits** - Set memory/CPU limits
5. **Monitoring** - Metrics and alerting
