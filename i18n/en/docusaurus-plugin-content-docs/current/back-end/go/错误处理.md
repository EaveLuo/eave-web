# Error Handling - The Go Way

Go doesn't have exceptions, only errors. This seemingly simple approach leads to clearer, more reliable code.

## 🎯 Errors as Values

In Go, an error is a value:

\`\`\`go
type error interface {
    Error() string
}

// Function returns error
func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

// Handle error
result, err := divide(10, 0)
if err != nil {
    fmt.Println("Error:", err)
    return
}
fmt.Println("Result:", result)
\`\`\`

## 🔧 Creating Errors

### 1. errors.New

\`\`\`go
err := errors.New("something went wrong")
\`\`\`

### 2. fmt.Errorf (Formatting)

\`\`\`go
name := "John"
err := fmt.Errorf("user %s not found", name)
\`\`\`

### 3. Custom Error Types

\`\`\`go
type ValidationError struct {
    Field   string
    Message string
}

func (e ValidationError) Error() string {
    return fmt.Sprintf("%s: %s", e.Field, e.Message)
}

// Usage
err := ValidationError{
    Field:   "email",
    Message: "invalid format",
}
\`\`\`

## 🎨 Error Handling Patterns

### 1. Direct Return

\`\`\`go
func process() error {
    if err := step1(); err != nil {
        return err
    }
    if err := step2(); err != nil {
        return err
    }
    return nil
}
\`\`\`

### 2. Wrap Errors

\`\`\`go
func loadConfig() error {
    data, err := os.Open("config.json")
    if err != nil {
        return fmt.Errorf("opening config file: %w", err)
    }
    // ...
}
\`\`\`

### 3. Sentinel Errors

\`\`\`go
var ErrNotFound = errors.New("not found")

func GetUser(id int) (*User, error) {
    if notFound {
        return nil, ErrNotFound
    }
    return user, nil
}

// Check specific error
user, err := GetUser(123)
if err == ErrNotFound {
    // Handle not found
}
\`\`\`

## 🔍 Error Checking Techniques

### 1. Early Return

\`\`\`go
// ✅ Clean
func process() error {
    if err := validate(); err != nil {
        return err
    }
    if err := load(); err != nil {
        return err
    }
    // Main logic
    return nil
}

// ❌ Deep nesting
func process2() error {
    if err := validate(); err == nil {
        if err := load(); err == nil {
            // Main logic
        }
    }
    return err
}
\`\`\`

### 2. Error Helper Functions

\`\`\`go
func check(err error) {
    if err != nil {
        log.Fatal(err)
    }
}

// Usage
file, err := os.Open("file.txt")
check(err)
\`\`\`

## 🎁 Error Wrapping and Unwrapping

### 1. Wrap Errors

\`\`\`go
err := os.Open("file.txt")
if err != nil {
    return fmt.Errorf("reading file failed: %w", err)
}
\`\`\`

### 2. Unwrap Errors

\`\`\`go
err := doSomething()

// Check specific error
if errors.Is(err, os.ErrNotExist) {
    fmt.Println("File not found")
}

// Extract specific error
var pathErr *os.PathError
if errors.As(err, &pathErr) {
    fmt.Println("Path error:", pathErr.Path)
}

// Unwrap
unwrapped := errors.Unwrap(err)
\`\`\`

## 🐛 Common Mistakes

### 1. Ignoring Errors

\`\`\`go
// ❌ Dangerous
file, _ := os.Open("file.txt")
// file may be nil!

// ✅ Correct
file, err := os.Open("file.txt")
if err != nil {
    // Handle error
}
\`\`\`

### 2. Unclear Error Messages

\`\`\`go
// ❌ Vague
if err != nil {
    return errors.New("failed")
}

// ✅ Clear
if err != nil {
    return fmt.Errorf("loading user %d data: %w", userID, err)
}
\`\`\`

### 3. Double Wrapping

\`\`\`go
// ❌ Over-wrapping
err := step1()
if err != nil {
    return fmt.Errorf("step1: %w", err)
}

err = step2()
if err != nil {
    return fmt.Errorf("step2: %w", fmt.Errorf("step1: %w", err))
}

// ✅ Wrap once per layer
\`\`\`

## 💡 Best Practices

### 1. Errors as Data

\`\`\`go
// ✅ Treat errors as return values
result, err := doSomething()
if err != nil {
    return err
}

// ❌ Don't use panic for normal errors
if somethingWrong {
    panic("something wrong")  // Unless truly exceptional
}
\`\`\`

### 2. Sentinel Errors

\`\`\`go
// Define sentinel errors
var ErrNotFound = errors.New("not found")

// Use
func GetUser(id int) (*User, error) {
    if notFound {
        return nil, ErrNotFound
    }
}

// Check
if err == ErrNotFound {
    // Specific handling
}
\`\`\`

### 3. Error Types

\`\`\`go
type HTTPError struct {
    Code   int
    Status string
}

func (e HTTPError) Error() string {
    return fmt.Sprintf("HTTP %d: %s", e.Code, e.Status)
}

// Use
func request() error {
    return HTTPError{Code: 404, Status: "Not Found"}
}

// Check
var httpErr HTTPError
if errors.As(err, &httpErr) {
    fmt.Println("HTTP error code:", httpErr.Code)
}
\`\`\`

### 4. Resource Cleanup

\`\`\`go
func process() error {
    file, err := os.Open("file.txt")
    if err != nil {
        return err
    }
    defer file.Close()  // Ensure close
    
    // Process file
}
\`\`\`

## ✅ Summary

- ✅ Errors are values, not exceptions
- ✅ Use errors.New or fmt.Errorf to create errors
- ✅ Use %w to wrap errors
- ✅ Use errors.Is and errors.As to check errors
- ✅ Early return, avoid nesting
- ✅ Use defer for resource cleanup
