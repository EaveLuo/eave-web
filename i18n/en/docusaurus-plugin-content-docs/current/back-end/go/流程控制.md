# Control Flow - Go's Style of if, for, switch

Go's control flow is concise yet powerful. No fancy syntax, just practical tools.

## ArrowRightHook if-else Conditional

### 1. Basic Usage

```go
age := 18

if age >= 18 {
    fmt.Println("Adult")
} else {
    fmt.Println("Minor")
}
```

### 2. if with Initialization Statement Star

This is Go's special syntax:

```go
// Execute initialization before condition check
if age := getAge(); age >= 18 {
    fmt.Println("Adult")
} else {
    fmt.Println("Minor")
}

// age variable is only valid within this if block
// fmt.Println(age)  // CrossMark Compile error
```

**Advantage:** Clear variable scope, avoids polluting outer scope.

### 3. Multiple Conditions

```go
score := 85

if score >= 90 {
    fmt.Println("Excellent")
} else if score >= 80 {
    fmt.Println("Good")
} else if score >= 60 {
    fmt.Println("Pass")
} else {
    fmt.Println("Fail")
}
```

### 4. No Parentheses in if

```go
// WhiteCheckMark Go style
if age >= 18 {
    // ...
}

// CrossMark No need for parentheses
if (age >= 18) {
    // ...
}
```

## RepeatButton for Loops

Go **only has for loops**, but they're powerful!

### 1. Standard for Loop

```go
// Standard form
for i := 0; i < 10; i++ {
    fmt.Println(i)
}
```

### 2. while Style

```go
// Omit init and post statements
i := 0
for i < 10 {
    fmt.Println(i)
    i++
}
```

### 3. Infinite Loop

```go
// Omit all conditions (infinite loop)
for {
    fmt.Println("Running forever...")
    time.Sleep(time.Second)
}
```

### 4. range Loop

Iterate over collections:

```go
// Array/Slice
nums := []int{1, 2, 3}
for i, v := range nums {
    fmt.Printf("Index %d: %d\n", i, v)
}

// Map
ages := map[string]int{"John": 25, "Jane": 30}
for name, age := range ages {
    fmt.Printf("%s is %d years old\n", name, age)
}

// String (iterates over runes)
for i, r := range "Hello" {
    fmt.Printf("Index %d: %c\n", i, r)
}

// Channel
for v := range ch {
    fmt.Println(v)
}
```

### 5. break and continue

```go
// break - exit loop
for i := 0; i < 10; i++ {
    if i == 5 {
        break  // Exit loop
    }
    fmt.Println(i)
}

// continue - skip to next iteration
for i := 0; i < 10; i++ {
    if i%2 == 0 {
        continue  // Skip even numbers
    }
    fmt.Println(i)  // Only odd numbers
}
```

### 6. Labels and Nested Loops

```go
OuterLoop:
    for i := 0; i < 3; i++ {
        for j := 0; j < 3; j++ {
            if i*j > 2 {
                break OuterLoop  // Break outer loop
            }
            fmt.Printf("%d * %d = %d\n", i, j, i*j)
        }
    }
```

## TwistedRightwardsArrows switch Statement

Go's switch is more flexible than C/Java!

### 1. Basic switch

```go
day := "Monday"

switch day {
case "Monday":
    fmt.Println("Start of work week")
case "Friday":
    fmt.Println("TGIF!")
case "Saturday", "Sunday":
    fmt.Println("Weekend!")
default:
    fmt.Println("Midweek")
}
```

### 2. No Automatic Fallthrough

```go
// In Go, cases don't fall through by default!
switch n := 2; n {
case 1:
    fmt.Println("One")  // Won't print
    // No need for 'break'
case 2:
    fmt.Println("Two")  // Prints
    // Automatically breaks here
case 3:
    fmt.Println("Three")  // Won't print
}
```

### 3. Explicit Fallthrough

```go
switch num := 1; num {
case 1:
    fmt.Println("One")
    fallthrough  // Continue to next case
case 2:
    fmt.Println("Two")  // Also prints!
case 3:
    fmt.Println("Three")
}
```

### 4. Switch without Expression

```go
score := 85

switch {
case score >= 90:
    fmt.Println("A")
case score >= 80:
    fmt.Println("B")
case score >= 70:
    fmt.Println("C")
case score >= 60:
    fmt.Println("D")
default:
    fmt.Println("F")
}
```

### 5. Type Switch

```go
var x interface{} = 42

switch v := x.(type) {
case int:
    fmt.Printf("Integer: %d\n", v)
case string:
    fmt.Printf("String: %s\n", v)
case bool:
    fmt.Printf("Boolean: %v\n", v)
default:
    fmt.Printf("Unknown type: %T\n", v)
}
```

## ArrowHeadingUp goto Statement

Go supports `goto`, but use sparingly!

```go
func main() {
    i := 0

Loop:
    if i < 5 {
        fmt.Println(i)
        i++
        goto Loop
    }
}
```

**Best practice:** Avoid goto when possible. Use it only for breaking out of deeply nested loops.

## Target Practical Examples

### Example 1: Find First Match

```go
func findFirst(nums []int, target int) (int, bool) {
    for i, v := range nums {
        if v == target {
            return i, true
        }
    }
    return -1, false
}
```

### Example 2: Retry with Timeout

```go
func retryWithTimeout(maxRetries int, timeout time.Duration) error {
    for i := 0; i < maxRetries; i++ {
        if err := doSomething(); err == nil {
            return nil
        }
        
        if i < maxRetries-1 {
            time.Sleep(timeout)
        }
    }
    return errors.New("max retries exceeded")
}
```

### Example 3: Process Until Done

```go
func processJobs(jobs <-chan Job) {
    for {
        select {
        case job, ok := <-jobs:
            if !ok {
                return  // Channel closed
            }
            process(job)
        case <-time.After(time.Minute):
            return  // Timeout
        }
    }
}
```

### Example 4: Fibonacci

```go
func fibonacci(n int) int {
    if n <= 1 {
        return n
    }
    
    a, b := 0, 1
    for i := 2; i <= n; i++ {
        a, b = b, a+b
    }
    return b
}
```

### Example 5: Grade Calculator

```go
func getGrade(score int) string {
    switch {
    case score >= 90:
        return "A"
    case score >= 80:
        return "B"
    case score >= 70:
        return "C"
    case score >= 60:
        return "D"
    default:
        return "F"
    }
}
```

## Bug Common Mistakes

### 1. Forgetting Braces

```go
// CrossMark Wrong
if age >= 18
    fmt.Println("Adult")

// WhiteCheckMark Correct
if age >= 18 {
    fmt.Println("Adult")
}
```

### 2. Variable Shadowing in if

```go
x := 10

if x := 5; x > 0 {
    fmt.Println(x)  // Prints 5 (shadowed variable)
}

fmt.Println(x)  // Prints 10 (original variable)
```

### 3. Modifying Range Variable

```go
nums := []int{1, 2, 3}

for _, n := range nums {
    n *= 2  // CrossMark Only modifies copy!
}

// WhiteCheckMark Correct way
for i := range nums {
    nums[i] *= 2
}
```

### 4. Infinite Loop Without Exit

```go
// CrossMark Forgot to increment
for i := 0; i < 10; {
    fmt.Println(i)
    // Forgot i++!
}
```

### 5. Misunderstanding Range Return Values

```go
nums := []int{1, 2, 3}

// CrossMark Redundant
for i := range nums {
    fmt.Println(nums[i])
}

// WhiteCheckMark Recommended
for _, num := range nums {
    fmt.Println(num)
}

// Or
for i, num := range nums {
    fmt.Printf("%d: %d\n", i, num)
}
```

### 6. Forgetting Default in switch

```go
// Warning consider adding default
switch status {
case "active":
    // ...
case "inactive":
    // ...
default:
    log.Printf("Unknown status: %s", status)
}
```

## LightBulb Best Practices

### 1. Simplify if Statements

```go
// CrossMark Redundant
if isValid == true {
    // ...
}

// WhiteCheckMark Concise
if isValid {
    // ...
}

// CrossMark Redundant
if isValid == false {
    // ...
}

// WhiteCheckMark Concise
if !isValid {
    // ...
}
```

### 2. Choose Right for Loop

```go
// Need index and value
for i, item := range items {
    // ...
}

// Only need value
for _, item := range items {
    // ...
}

// Need to modify elements
for i := range items {
    items[i] = transform(items[i])
}
```

### 3. Align switch Cases

```go
// WhiteCheckMark Clean
switch status {
case "active":
    handleActive()
case "pending":
    handlePending()
case "closed":
    handleClosed()
default:
    logUnknown()
}
```

### 4. Early Returns

```go
// CrossMark Too nested
func process(data []int) error {
    if data != nil {
        if len(data) > 0 {
            if data[0] > 0 {
                // Processing logic
            }
        }
    }
    return nil
}

// WhiteCheckMark Early returns
func process(data []int) error {
    if data == nil {
        return nil
    }
    if len(data) == 0 {
        return nil
    }
    if data[0] <= 0 {
        return nil
    }
    // Processing logic
    return nil
}
```

### 5. Prefer Early Returns

```go
// WhiteCheckMark Clean
func process(data Data) error {
    if data.Invalid() {
        return errors.New("invalid data")
    }
    
    if data.Empty() {
        return errors.New("empty data")
    }
    
    return doProcess(data)
}

// CrossMark Nested
func process(data Data) error {
    if data.Valid() {
        if !data.Empty() {
            return doProcess(data)
        } else {
            return errors.New("empty data")
        }
    } else {
        return errors.New("invalid data")
    }
}
```

### 6. Use range for Collections

```go
// WhiteCheckMark Idiomatic
for i, v := range slice {
    // ...
}

// CrossMark Manual indexing
for i := 0; i < len(slice); i++ {
    v := slice[i]
    // ...
}
```

### 7. Leverage switch for Multiple Cases

```go
// WhiteCheckMark Clean switch
switch cmd {
case "start", "run":
    start()
case "stop", "halt":
    stop()
case "restart", "reload":
    restart()
}

// CrossMark If-else chain
if cmd == "start" || cmd == "run" {
    start()
} else if cmd == "stop" || cmd == "halt" {
    stop()
} // ...
```

## WhiteCheckMark Key Points Summary

- WhiteCheckMark if can have initialization statement
- WhiteCheckMark No parentheses needed around conditions
- WhiteCheckMark for is the only loop construct
- WhiteCheckMark range iterates over collections
- WhiteCheckMark switch doesn't fall through by default
- WhiteCheckMark Use labels for breaking nested loops
- WhiteCheckMark Avoid goto unless necessary
- WhiteCheckMark Use early returns to reduce nesting
- WhiteCheckMark Prefer range over manual indexing
