# Pointers - 理解 Go 的内存模型

Pointers是 Go 中强大但常被误解的特性。理解Pointers，是掌握 Go 内存模型的关键。

## 📍 什么是Pointers？

Pointers存储变量的**内存地址**，而不是值本身。

```go
a := 42
p := &a  // p 指向 a 的地址

fmt.Println(a)   // 42 - 值
fmt.Println(&a)  // 0xc000014090 - 地址
fmt.Println(p)   // 0xc000014090 - Pointers
fmt.Println(*p)  // 42 - 解引用，获取值
```

## 🔑 两个关键操作符

### 1. & 取地址

```go
name := "张三"
p := &name  // p 是 name 的地址
```

### 2. * 解引用

```go
p := &name
fmt.Println(*p)  // "张三" - 获取 p 指向的值

// 修改值
*p = "李四"
fmt.Println(name)  // "李四" - name 也被修改了
```

## 💡 为什么要用Pointers？

### 1. 修改函数参数

```go
// ❌ Error：无法修改原值
func double(x int) {
    x = x * 2
}

num := 5
double(num)
fmt.Println(num)  // 5 - 没变！

// ✅ correct：使用Pointers
func doublePtr(p *int) {
    *p = *p * 2
}

num = 5
doublePtr(&num)
fmt.Println(num)  // 10 - 变了！
```

### 2. 避免大对象复制

```go
type LargeStruct struct {
    data [1000]int
}

// ❌ 低效：复制整个结构体
func process(s LargeStruct) {
    // ...
}

// ✅ 高效：只传递Pointers
func processPtr(s *LargeStruct) {
    // ...
}
```

### 3. 表示"无值"

```go
// Pointers可以是 nil，表示"无值"
var p *int = nil

if p == nil {
    fmt.Println("没有值")
}
```

## 📊 Pointers的Pointers

```go
a := 42
p1 := &a   // *int
p2 := &p1  // **int

fmt.Println(*p1)   // 42
fmt.Println(**p2)  // 42
```

## 🎯 Pointers与数据结构

### 1. Pointers与数组

```go
arr := [3]int{1, 2, 3}
p := &arr

fmt.Println(p[0])  // 1 - 可以直接索引
p[1] = 20
fmt.Println(arr)   // [1 20 3]
```

### 2. Pointers与切片

```go
// 切片本身包含Pointers，通常不需要再传Pointers
func modify(slice []int) {
    slice[0] = 100
}

s := []int{1, 2, 3}
modify(s)
fmt.Println(s)  // [100 2 3]
```

### 3. Pointers与 Map

```go
// Map 也是引用类型，不需要Pointers
func update(m map[string]int) {
    m["key"] = 42
}

m := make(map[string]int)
update(m)
```

## 🐛 Common Mistakes

### 1. 空Pointers解引用

```go
var p *int
fmt.Println(*p)  // ❌ panic!

// ✅ 检查 nil
if p != nil {
    fmt.Println(*p)
}
```

### 2. 返回局部变量Pointers

```go
// ❌ Error（但在 Go 中是安全的，因为逃逸分析）
func getPointer() *int {
    x := 42
    return &x  // Go 会自动分配到堆上
}

// ✅ 推荐：明确分配
func getPointer2() *int {
    return new(int)
}
```

### 3. 混淆Pointers和值

```go
type Person struct {
    Name string
}

p := Person{Name: "张三"}

// ❌ Error
var ptr *Person = p  // 类型不匹配

// ✅ correct
var ptr *Person = &p
```

## 💡 Best Practices

### 1. 何时使用Pointers

```go
// ✅ 使用Pointers的场景
- 需要修改函数参数
- 大结构体（避免复制）
- 需要表示"无值"（nil）
- Methods需要修改Receive者

// ✅ 使用值的场景
- 小类型（int, bool, string）
- 不需要修改
- 切片、Map、Channel（本身是引用）
```

### 2. MethodsReceive者选择

```go
type Counter struct {
    count int
}

// 值Receive者：不修改原对象
func (c Counter) Get() int {
    return c.count
}

// PointersReceive者：修改原对象
func (c *Counter) Increment() {
    c.count++
}
```

### 3. 一致性原则

```go
// ✅ 保持一致：如果有一个Methods用Pointers，都用Pointers
type Config struct {
    value int
}

func (c *Config) Get() int     { return c.value }
func (c *Config) Set(v int)    { c.value = v }
func (c *Config) Reset()       { c.value = 0 }

// ⚠️ 避免混用
func (c Config) Get2() int     // 不一致
```

## ✅ Summary

- ✅ `&` 取地址，`*` 解引用
- ✅ Pointers可以修改函数参数
- ✅ Pointers避免大对象复制
- ✅ Pointers可以是 nil
- ✅ 切片、Map、Channel 是引用类型
- ✅ MethodsReceive者选择Pointers或值
