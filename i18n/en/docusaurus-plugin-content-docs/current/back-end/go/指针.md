# Pointers - Understanding Go's Memory Model

Pointers are a powerful but often misunderstood feature in Go. Understanding pointers is key to mastering Go's memory model.

## RoundPushpin What is a Pointer?

A pointer stores the **memory address** of a variable, not the value itself.

```go
a := 42
p := &a  // p points to a's address

fmt.Println(a)   // 42 - value
fmt.Println(&a)  // 0xc000014090 - address
fmt.Println(p)   // 0xc000014090 - pointer
fmt.Println(*p)  // 42 - dereference to get value
```

## Key Two Key Operators

### 1. & Address-of

```go
name := "John"
p := &name  // p is the address of name
```

### 2. * Dereference

```go
p := &name
fmt.Println(*p)  // "John" - get the value p points to

// Modify value
*p = "Jane"
fmt.Println(name)  // "Jane" - name is also modified
```

## LightBulb Why Use Pointers?

### 1. Modify Function Parameters

```go
// CrossMark Wrong: can't modify original value
func double(x int) {
    x = x * 2
}

num := 5
double(num)
fmt.Println(num)  // 5 - unchanged!

// WhiteCheckMark Correct: use pointer
func doublePtr(p *int) {
    *p = *p * 2
}

num = 5
doublePtr(&num)
fmt.Println(num)  // 10 - changed!
```

### 2. Avoid Large Object Copying

```go
type LargeStruct struct {
    data [1000]int
}

// CrossMark Inefficient: copies entire struct
func process(s LargeStruct) {
    // ...
}

// WhiteCheckMark Efficient: only passes pointer
func processPtr(s *LargeStruct) {
    // ...
}
```

### 3. Share Data Between Functions

```go
func main() {
    counter := 0
    
    // Multiple goroutines share the same counter
    go increment(&counter)
    go increment(&counter)
    
    time.Sleep(time.Second)
    fmt.Println(counter)  // 2
}

func increment(p *int) {
    *p++
}
```

## Warning Common Mistakes

### 1. Nil Pointer Dereference

```go
var p *int  // nil pointer
fmt.Println(*p)  // CrossMark Panic: runtime error: invalid memory address

// WhiteCheckMark Always check for nil
if p != nil {
    fmt.Println(*p)
}
```

### 2. Returning Pointer to Local Variable

```go
// CrossMark Dangerous (but Go handles this!)
func bad() *int {
    x := 42
    return &x  // Go escapes x to heap
}

// This actually works in Go due to escape analysis
```

### 3. Confusing * and &

```go
x := 42
p := &x   // p is *int (pointer)

// CrossMark Wrong
value := &p   // This is **int (pointer to pointer)

// WhiteCheckMark Correct
value := *p   // This is int (dereferenced value)
```

## WhiteCheckMark Best Practices

### 1. When to Use Pointers

Use pointers when you need to:
- Modify the original value
- Avoid copying large structures
- Share data between functions
- Represent optional values (nil)

### 2. When NOT to Use Pointers

Don't use pointers for:
- Small primitive types (int, bool, etc.)
- Values that shouldn't be modified
- Slices and maps (they're already reference types)

### 3. Pointer vs Value Receiver

```go
type User struct {
    Name string
    Age  int
}

// Value receiver - doesn't modify original
func (u User) GetName() string {
    return u.Name
}

// Pointer receiver - can modify original
func (u *User) Birthday() {
    u.Age++
}
```

### 4. Check for Nil

```go
func process(p *Data) error {
    if p == nil {
        return errors.New("nil pointer")
    }
    // Process...
    return nil
}
```

## ArrowsClockwise new() and make()

Both allocate memory, but for different types:

### new() - Allocate Zero Value

```go
// Allocates memory, returns pointer
p := new(int)      // *int, value is 0
s := new(string)   // *string, value is ""

// Equivalent to:
var i int
p = &i
```

### make() - Initialize Built-in Types

```go
// For slices, maps, channels only
s := make([]int, 10)       // slice
m := make(map[string]int)  // map
ch := make(chan int)       // channel
```

**Key difference:**
- `new()` allocates zeroed memory, returns pointer
- `make()` initializes and returns the type itself (not pointer)

## Target Practical Examples

### Example 1: Swap Function

```go
func swap(a, b *int) {
    *a, *b = *b, *a
}

x, y := 10, 20
swap(&x, &y)
fmt.Println(x, y)  // 20 10
```

### Example 2: Linked List Node

```go
type Node struct {
    Value int
    Next  *Node
}

// Create linked list
head := &Node{Value: 1}
head.Next = &Node{Value: 2}
head.Next.Next = &Node{Value: 3}

// Traverse
for n := head; n != nil; n = n.Next {
    fmt.Println(n.Value)
}
```

### Example 3: Optional Configuration

```go
type Config struct {
    Host *string  // Optional - nil means use default
    Port *int     // Optional - nil means use default
}

func applyConfig(c *Config) {
    host := "localhost"
    if c.Host != nil {
        host = *c.Host
    }
    
    port := 8080
    if c.Port != nil {
        port = *c.Port
    }
    
    fmt.Printf("Connecting to %s:%d\n", host, port)
}
```

## WhiteCheckMark Key Points Summary

- WhiteCheckMark Pointers store memory addresses, not values
- WhiteCheckMark `&` gets address, `*` dereferences
- WhiteCheckMark Use pointers to modify originals and avoid copying
- WhiteCheckMark Always check for nil before dereferencing
- WhiteCheckMark `new()` allocates, `make()` initializes
- WhiteCheckMark Slices, maps, channels are already reference types
