# 指针 - 理解 Go 的内存模型

指针是 Go 中强大但常被误解的特性。理解指针，是掌握 Go 内存模型的关键。

## 📍 What is a Pointer？

指针存储变量的**内存地址**，而不是值本身。

```go
a := 42
p := &a  // p 指向 a 的地址

fmt.Println(a)   // 42 - 值
fmt.Println(&a)  // 0xc000014090 - 地址
fmt.Println(p)   // 0xc000014090 - 指针
fmt.Println(*p)  // 42 - 解引用，获取值
```

## 🔑 两个关键操作符

### 1. & 取地址

```go
name := "张三"
p := &name  // p 是 name 的地址
```

### 2. * 解引用

```go
p := &name
fmt.Println(*p)  // "张三" - 获取 p 指向的值

// 修改值
*p = "李四"
fmt.Println(name)  // "李四" - name 也被修改了
```

## 💡 为什么要用指针？

### 1. 修改函数参数

```go
// ❌ 错误：无法修改原值
func double(x int) {
    x = x * 2
}

num := 5
double(num)
fmt.Println(num)  // 5 - 没变！

// ✅ 正确：使用指针
func doublePtr(p *int) {
    *p = *p * 2
}

num = 5
doublePtr(&num)
fmt.Println(num)  // 10 - 变了！
```

### 2. 避免大对象复制

```go
type LargeStruct struct {
    data [1000]int
}

// ❌ 低效：复制整个结构体
func process(s LargeStruct) {
    // ...
}

// ✅ 高效：只传递指针
func processPtr(s *LargeStruct) {
    // ...
}
```

### 3. 表示"无值"

```go
// 指针可以是 nil，表示"无值"
var p *int = nil

if p == nil {
    fmt.Println("没有值")
}
```

## 📊 指针的指针

```go
a := 42
p1 := &a   // *int
p2 := &p1  // **int

fmt.Println(*p1)   // 42
fmt.Println(**p2)  // 42
```

## 🎯 指针与数据结构

### 1. 指针与数组

```go
arr := [3]int{1, 2, 3}
p := &arr

fmt.Println(p[0])  // 1 - 可以直接索引
p[1] = 20
fmt.Println(arr)   // [1 20 3]
```

### 2. 指针与切片

```go
// 切片本身包含指针，通常不需要再传指针
func modify(slice []int) {
    slice[0] = 100
}

s := []int{1, 2, 3}
modify(s)
fmt.Println(s)  // [100 2 3]
```

### 3. 指针与 Map

```go
// Map 也是引用类型，不需要指针
func update(m map[string]int) {
    m["key"] = 42
}

m := make(map[string]int)
update(m)
```

## 🐛 Common Mistakes

### 1. 空指针解引用

```go
var p *int
fmt.Println(*p)  // ❌ panic!

// ✅ 检查 nil
if p != nil {
    fmt.Println(*p)
}
```

### 2. 返回局部变量指针

```go
// ❌ 错误（但在 Go 中是安全的，因为逃逸分析）
func getPointer() *int {
    x := 42
    return &x  // Go 会自动分配到堆上
}

// ✅ 推荐：明确分配
func getPointer2() *int {
    return new(int)
}
```

### 3. 混淆指针和值

```go
type Person struct {
    Name string
}

p := Person{Name: "张三"}

// ❌ 错误
var ptr *Person = p  // 类型不匹配

// ✅ 正确
var ptr *Person = &p
```

## 💡 Best Practices

### 1. 何时使用指针

```go
// ✅ 使用指针的场景
- 需要修改函数参数
- 大结构体（避免复制）
- 需要表示"无值"（nil）
- 方法需要修改接收者

// ✅ 使用值的场景
- 小类型（int, bool, string）
- 不需要修改
- 切片、Map、Channel（本身是引用）
```

### 2. 方法Receive者选择

```go
type Counter struct {
    count int
}

// 值接收者：不修改原对象
func (c Counter) Get() int {
    return c.count
}

// 指针接收者：修改原对象
func (c *Counter) Increment() {
    c.count++
}
```

### 3. 一致性原则

```go
// ✅ 保持一致：如果有一个方法用指针，都用指针
type Config struct {
    value int
}

func (c *Config) Get() int     { return c.value }
func (c *Config) Set(v int)    { c.value = v }
func (c *Config) Reset()       { c.value = 0 }

// ⚠️ 避免混用
func (c Config) Get2() int     // 不一致
```

## ✅ Summary

- ✅ `&` 取地址，`*` 解引用
- ✅ 指针可以修改函数参数
- ✅ 指针避免大对象复制
- ✅ 指针可以是 nil
- ✅ 切片、Map、Channel 是引用类型
- ✅ 方法Receive者选择指针或值
