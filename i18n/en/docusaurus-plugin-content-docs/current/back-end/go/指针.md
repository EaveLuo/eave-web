# Pointers - Understanding Go's Memory Model

Pointers are powerful but often misunderstood in Go. Understanding pointers is key to mastering Go's memory model.

## 📍 What is a Pointer?

A pointer stores the **memory address** of a variable, not the value itself.

```go
a := 42
p := &a  // p points to a's address

fmt.Println(a)   // 42 - value
fmt.Println(&a)  // 0xc000014090 - address
fmt.Println(p)   // 0xc000014090 - pointer
fmt.Println(*p)  // 42 - dereference to get value
```

## 🔑 Two Key Operators

### 1. & Address-of

```go
name := "John"
p := &name  // p is the address of name
```

### 2. * Dereference

```go
p := &name
fmt.Println(*p)  // "John" - get value at p

// Modify value through pointer
*p = "Jane"
fmt.Println(name)  // "Jane" - name is also changed
```

## 💡 Why Use Pointers?

### 1. Modify Function Parameters

```go
// ❌ Wrong: Can't modify original value
func double(x int) {
    x = x * 2
}

num := 5
double(num)
fmt.Println(num)  // 5 - unchanged!

// ✅ Correct: Use pointer
func doublePtr(p *int) {
    *p = *p * 2
}

num := 5
doublePtr(&num)
fmt.Println(num)  // 10 - modified!
```

### 2. Avoid Copying Large Data

```go
type BigStruct struct {
    data [10000]int
}

// ❌ Bad: Copies entire struct (slow)
func process(s BigStruct) {
    // s is a copy
}

// ✅ Good: Only copies pointer (fast)
func processPtr(s *BigStruct) {
    // s points to original
}
```

### 3. Share Data Between Functions

```go
func main() {
    counter := 0
    
    increment(&counter)
    increment(&counter)
    
    fmt.Println(counter)  // 2
}

func increment(p *int) {
    *p++
}
```

## 🆚 Value vs Pointer Receivers

### Value Receiver (Copy)

```go
type Person struct {
    Name string
    Age  int
}

func (p Person) Birthday() {
    p.Age++  // Modifies copy only!
}

person := Person{Name: "John", Age: 25}
person.Birthday()
fmt.Println(person.Age)  // 25 - unchanged
```

### Pointer Receiver (Modify Original)

```go
func (p *Person) Birthday() {
    p.Age++  // Modifies original!
}

person := Person{Name: "John", Age: 25}
person.Birthday()
fmt.Println(person.Age)  // 26 - changed!
```

**When to use pointer receivers:**
- Need to modify the receiver
- Large structs (avoid copying)
- Consistency (if some methods need pointer, use for all)

## ⚠️ Common Pitfalls

### 1. Nil Pointer Dereference

```go
var p *int  // nil pointer
fmt.Println(*p)  // ❌ Panic: runtime error

// Always check before dereferencing
if p != nil {
    fmt.Println(*p)
}
```

### 2. Returning Address of Local Variable

```go
// ✅ Safe: Go escapes variable to heap
func newInt() *int {
    i := 42
    return &i  // OK in Go!
}

p := newInt()
fmt.Println(*p)  // 42
```

**Go's escape analysis** automatically moves variables to heap when needed.

### 3. Pointer to Pointer

```go
a := 10
p := &a   // pointer to int
pp := &p  // pointer to pointer

fmt.Println(**pp)  // 10 - double dereference
```

Rarely needed in practice.

## 🎯 new() Function

`new()` allocates memory and returns a pointer:

```go
// These are equivalent
p1 := new(int)     // *int, zero value 0
p2 := &int{}       // Same as above

// Initialize with value
p3 := new(int)
*p3 = 42

// For structs
person := new(Person)  // *Person
person.Name = "John"
```

## 💡 Best Practices

### 1. Use Pointers Judiciously

```go
// ✅ Good: Small struct, use value
type Point struct{ X, Y float64 }
func move(p Point, dx, dy float64) Point

// ✅ Good: Large struct or needs modification
type Image struct { /* lots of data */ }
func (img *Image) Resize(width, height int)
```

### 2. Check for Nil

```go
func process(p *Person) error {
    if p == nil {
        return errors.New("nil person")
    }
    // Process...
    return nil
}
```

### 3. Document Pointer Behavior

```go
// Update modifies the user in place.
// The caller's User object will be modified.
func (u *User) Update(name string) error
```

## 📊 Summary

| Concept | Syntax | Meaning |
|---------|--------|---------|
| Address-of | `&x` | Get address of x |
| Dereference | `*p` | Get value at p |
| Pointer type | `*int` | Pointer to int |
| Nil pointer | `var p *T` | Uninitialized pointer |

**Remember:**
- Pointers allow modifying original values
- Pointers avoid copying large data
- Always check for nil before dereferencing
- Go's garbage collector handles memory automatically
