# Structs - A New Approach to Object-Oriented Programming

Go doesn't have classes, but uses structs to achieve similar encapsulation. Let's explore Go-style "object-oriented programming".

## 📦 Defining Structs

```go
type Person struct {
    Name string
    Age  int
    City string
}
```

## 🎯 Creating Instances

```go
// Method 1: Literal
p1 := Person{Name: "John", Age: 25, City: "Beijing"}

// Method 2: Partial fields (others are zero values)
p2 := Person{Name: "Jane"}  // Age=0, City=""

// Method 3: By order (not recommended)
p3 := Person{"Bob", 30, "Shanghai"}

// Method 4: Using new (returns pointer)
p4 := new(Person)
p4.Name = "Alice"
```

## 🔧 Access and Modify

```go
p := Person{Name: "John", Age: 25}

// Access field
fmt.Println(p.Name)  // John

// Modify field
p.Age = 26

// Pointer access (automatic dereference)
pp := &p
fmt.Println(pp.Name)  // John (equivalent to (*pp).Name)
```

## 🔗 Embedded Fields (Composition)

Go uses composition instead of inheritance:

```go
type Address struct {
    City    string
    Country string
}

type Person struct {
    Name    string
    Age     int
    Address // Embedded field
}

p := Person{
    Name: "John",
    Age:  25,
    Address: Address{
        City:    "Beijing",
        Country: "China",
    },
}

// Access embedded fields directly
fmt.Println(p.City)    // Beijing
fmt.Println(p.Country) // China
```

## 🏷️ Struct Tags

Tags provide metadata for reflection:

```go
type User struct {
    ID       int    `json:"id" db:"user_id"`
    Name     string `json:"name" validate:"required"`
    Email    string `json:"email,omitempty"`
    Password string `json:"-"` // Ignored
}

// JSON marshaling uses tags
user := User{ID: 1, Name: "John", Email: "john@example.com"}
data, _ := json.Marshal(user)
// {"id":1,"name":"John","email":"john@example.com"}
```

## 💡 Best Practices

### 1. Use named fields

```go
// ✅ Good: Clear and explicit
p := Person{Name: "John", Age: 25}

// ❌ Bad: Unclear without context
p := Person{"John", 25}
```

### 2. Prefer pointers for large structs

```go
// ✅ Good: Avoid copying large structs
func processUser(u *User) error

// ⚠️ Okay for small structs
func updateName(u User) User
```

### 3. Use constructor functions

```go
// Constructor with validation
func NewUser(name, email string) (*User, error) {
    if name == "" {
        return nil, errors.New("name required")
    }
    return &User{Name: name, Email: email}, nil
}
```

## 📊 Comparison with Classes

| Feature | Class (OOP) | Struct (Go) |
|---------|-------------|-------------|
| Inheritance | Supported | Composition only |
| Methods | On class | On any type |
| Encapsulation | public/private | Capitalization |
| Constructors | Special methods | Factory functions |

**Go philosophy:** Composition over inheritance, simplicity over complexity.
