# Performance Optimization - Writing Efficient Go Code

Performance optimization needs data support. Learn to use tools, find bottlenecks, and optimize effectively.

## MagnifyingGlassTiltedLeft Profiling Tools

### 1. Benchmark Testing

```go
// benchmark_test.go
func BenchmarkStringConcat(b *testing.B) {
    for i := 0; i < b.N; i++ {
        s := ""
        for j := 0; j < 100; j++ {
            s += "hello"
        }
    }
}

func BenchmarkStringBuilder(b *testing.B) {
    for i := 0; i < b.N; i++ {
        var builder strings.Builder
        for j := 0; j < 100; j++ {
            builder.WriteString("hello")
        }
        _ = builder.String()
    }
}

// Run
// go test -bench=. -benchmem
```

### 2. CPU Profiling

```go
// main.go
import (
    "runtime/pprof"
    "os"
)

func main() {
    f, _ := os.Create("cpu.prof")
    pprof.StartCPUProfile(f)
    defer pprof.StopCPUProfile()
    
    // Program logic
}

// Analyze
// go tool pprof cpu.prof
```

### 3. Memory Profiling

```go
import (
    "runtime"
    "runtime/pprof"
)

func memProfile() {
    runtime.GC()
    f, _ := os.Create("mem.prof")
    pprof.WriteHeapProfile(f)
    f.Close()
}
```

## HighVoltage Optimization Techniques

### 1. String Concatenation

```go
// CrossMark Inefficient
s := ""
for i := 0; i < 1000; i++ {
    s += strconv.Itoa(i)
}

// WhiteCheckMark Efficient
var builder strings.Builder
for i := 0; i < 1000; i++ {
    builder.WriteString(strconv.Itoa(i))
}
s := builder.String()

// Or
parts := make([]string, 1000)
for i := 0; i < 1000; i++ {
    parts[i] = strconv.Itoa(i)
}
s := strings.Join(parts, "")
```

### 2. Slice Pre-allocation

```go
// CrossMark Frequent expansion
slice := []int{}
for i := 0; i < 10000; i++ {
    slice = append(slice, i)
}

// WhiteCheckMark Pre-allocate
slice := make([]int, 0, 10000)
for i := 0; i < 10000; i++ {
    slice = append(slice, i)
}
```

### 3. Map Pre-allocation

```go
// CrossMark Frequent expansion
m := make(map[string]int)
for i := 0; i < 10000; i++ {
    m[strconv.Itoa(i)] = i
}

// WhiteCheckMark Pre-allocate
m := make(map[string]int, 10000)
for i := 0; i < 10000; i++ {
    m[strconv.Itoa(i)] = i
}
```

### 4. Avoid Unnecessary Conversions

```go
// CrossMark Repeated conversion
s := "hello"
for i := 0; i < len(s); i++ {
    b := []byte(s)  // Convert every time
    b[i] = 'H'
}

// WhiteCheckMark Single conversion
b := []byte(s)
for i := 0; i < len(b); i++ {
    b[i] = 'H'
}
```

### 5. Use Arrays Instead of Slices

```go
// CrossMark Slice has extra overhead
func process(slice []int) {
    // ...
}

// WhiteCheckMark Array (when size known)
func processArray(arr [10]int) {
    // ...
}
```

### 6. Reduce Pointer Usage

```go
// CrossMark Small objects with pointers
func process(p *int) {
    // ...
}

// WhiteCheckMark Small objects by value
func process2(v int) {
    // ...
}
```

### 7. Sync Pool

```go
var bufferPool = sync.Pool{
    New: func() interface{} {
        return new(bytes.Buffer)
    },
}

func useBuffer() {
    buf := bufferPool.Get().(*bytes.Buffer)
    defer bufferPool.Put(buf)
    
    buf.Reset()
    // Use buf
}
```

## Target Concurrency Optimization

### 1. Limit Concurrent Goroutines

```go
// CrossMark Unlimited
for i := 0; i < 10000; i++ {
    go process(i)
}

// WhiteCheckMark Limit concurrency
sem := make(chan struct{}, 100)
for i := 0; i < 10000; i++ {
    sem <- struct{}{}
    go func(id int) {
        defer func() { <-sem }()
        process(id)
    }(i)
}
```

### 2. Batch Processing

```go
// CrossMark Individual processing
for _, item := range items {
    process(item)
}

// WhiteCheckMark Batch processing
batchSize := 100
for i := 0; i < len(items); i += batchSize {
    end := i + batchSize
    if end > len(items) {
        end = len(items)
    }
    
    batch := items[i:end]
    processBatch(batch)
}
```

### 3. Lock-free Design

```go
// CrossMark Lock contention
var mu sync.Mutex
var count int

func increment() {
    mu.Lock()
    count++
    mu.Unlock()
}

// WhiteCheckMark Atomic operations
var count2 int32

func increment2() {
    atomic.AddInt32(&count2, 1)
}
```

## BarChart Memory Optimization

### 1. Object Reuse

```go
// CrossMark Frequent allocation
func process() {
    data := make([]byte, 1024)
    // Use
}

// WhiteCheckMark Reuse
var pool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

func process2() {
    data := pool.Get().([]byte)
    defer pool.Put(data)
    // Use
}
```

### 2. Reduce GC Pressure

```go
// CrossMark Many short-lived objects
for i := 0; i < 1000000; i++ {
    s := fmt.Sprintf("item%d", i)
    process(s)
}

// WhiteCheckMark Reuse buffer
var buf []byte
for i := 0; i < 1000000; i++ {
    buf = buf[:0]
    buf = strconv.AppendInt(buf, int64(i), 10)
    process(string(buf))
}
```

### 3. Zero Value Initialization

```go
// CrossMark Unnecessary initialization
type Config struct {
    Name string
    Age  int
}

c := Config{Name: "", Age: 0}  // Redundant

// WhiteCheckMark Use zero value
c := Config{}  // Or var c Config
```

## LightBulb Best Practices

### 1. Correct First, Optimize Later

```go
// WhiteCheckMark Write correct code first
func process(data []int) []int {
    result := make([]int, len(data))
    for i, v := range data {
        result[i] = v * 2
    }
    return result
}

// Then optimize
// 1. Benchmark to find bottleneck
// 2. Optimize specific areas
// 3. Verify optimization effect
```

### 2. 80/20 Rule

```
80% of performance problems come from 20% of the code
```

**Approach:**
1. Use profiling to find hotspots
2. Optimize the most time-consuming 20% of code
3. Don't optimize prematurely

### 3. Continuous Monitoring

```go
// Add monitoring
import _ "net/http/pprof"

func main() {
    go func() {
        http.ListenAndServe("localhost:6060", nil)
    }()
    
    // Program logic
}

// Access http://localhost:6060/debug/pprof/
```
