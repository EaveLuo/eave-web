# Performance Optimization

Tips for writing high-performance Go code.

## 📊 Profiling

```bash
# CPU profile
go test -cpuprofile=cpu.prof -bench=.
go tool pprof cpu.prof

# Memory profile
go test -memprofile=mem.prof -bench=.
go tool pprof mem.prof
```

## 🚀 Optimization Tips

### 1. Pre-allocate Slices

```go
// ❌ Bad: Multiple allocations
var s []int
for i := 0; i < 1000; i++ {
    s = append(s, i)
}

// ✅ Good: Single allocation
s := make([]int, 0, 1000)
for i := 0; i < 1000; i++ {
    s = append(s, i)
}
```

### 2. Use sync.Pool

```go
var pool = sync.Pool{
    New: func() interface{} {
        return make([]byte, 1024)
    },
}

// Get from pool
buf := pool.Get().([]byte)
// Use...
pool.Put(buf)  // Return to pool
```

### 3. Avoid Allocations in Hot Paths

```go
// Reuse buffers
var buf bytes.Buffer
buf.Reset()
buf.WriteString(data)
```

## 💡 Best Practices

1. **Profile first** - Don't optimize prematurely
2. **Benchmark changes** - Measure impact
3. **Trade-offs** - Readability vs performance
