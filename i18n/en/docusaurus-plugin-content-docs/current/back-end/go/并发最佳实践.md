# 并发Best Practices - 避免常见陷阱

## ⚠️ 常见并发陷阱

### 1. Goroutine leak

```go
// ❌ Error：Goroutine leak
func worker(ch <-chan int) {
    for v := range ch {
        process(v)
    }
}

func main() {
    ch := make(chan int)
    go worker(ch)
    ch <- 1
    // 忘记Close channel，worker 永远阻塞
}

// ✅ correct
close(ch)  // Close channel，worker 退出
```

### 2. Race Conditions

```go
// ❌ Error：Race Conditions
var counter int
go func() { counter++ }()
go func() { counter++ }()

// ✅ correct：使用 Mutex
var mu sync.Mutex
var counter int
mu.Lock()
counter++
mu.Unlock()

// ✅ correct：使用 atomic
var counter int64
atomic.AddInt64(&counter, 1)
```

### 3. 死锁

```go
// ❌ Error：死锁
ch1 := make(chan int)
ch2 := make(chan int)
go func() { ch1 <- 1; <-ch2 }()
go func() { ch2 <- 2; <-ch1 }()

// ✅ correct：使用 select
select {
case ch1 <- 1:
    <-ch2
case <-time.After(time.Second):
    // timeout处理
}
```

## 🎯 Concurrency Patterns

### Worker Pool

```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for job := range jobs {
        results <- job * 2
    }
}

// 启动工作池
for w := 1; w <= 3; w++ {
    go worker(w, jobs, results)
}
```

## 📋 检查清单

- [ ] 所有 Goroutine 都能正常退出
- [ ] Channel 在适当时机Close
- [ ] 没有死锁风险
- [ ] 共享变量有适当保护
- [ ] 使用 context 控制timeout
