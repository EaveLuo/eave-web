# Concurrency Best Practices - Avoiding Common Pitfalls

## ⚠️ Common Pitfalls

### 1. Goroutine Leaks

```go
// ❌ Bad: Goroutine never exits
func worker(ch <-chan int) {
    for v := range ch {
        process(v)
    }
}

// Must close channel to stop worker
close(ch)
```

### 2. Race Conditions

```go
// ❌ Bad: Concurrent access without synchronization
var counter int

go func() { counter++ }()
go func() { counter++ }()

// ✅ Good: Use sync.Mutex or atomic
var mu sync.Mutex
mu.Lock()
counter++
mu.Unlock()
```

### 3. Deadlocks

```go
// ❌ Bad: Circular waiting
ch1 := make(chan int)
ch2 := make(chan int)

go func() { ch1 <- <-ch2 }()
go func() { ch2 <- <-ch1 }()
```

## 🎯 Patterns

### Worker Pool

```go
func worker(id int, jobs <-chan int, results chan<- int) {
    for j := range jobs {
        results <- process(j)
    }
}

// Start workers
for w := 1; w <= 3; w++ {
    go worker(w, jobs, results)
}
```

### Pipeline

```go
// Stage 1
gen := func(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums {
            out <- n
        }
        close(out)
    }()
    return out
}

// Stage 2
sq := func(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in {
            out <- n * n
        }
        close(out)
    }()
    return out
}
```

## 📋 Checklist

- [ ] All goroutines can exit cleanly
- [ ] Channels closed appropriately
- [ ] No race conditions (use -race flag)
- [ ] Context used for cancellation
- [ ] Proper error propagation
