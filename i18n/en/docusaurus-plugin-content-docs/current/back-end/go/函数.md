# Functions - Go's Core Building Blocks

Functions are the basic building blocks of Go programs. Go functions are concise yet powerful, supporting multiple return values, variadic parameters, and more.

## 📦 Function Definition

### 1. Basic Syntax

```go
// No parameters, no return value
func sayHello() {
    fmt.Println("Hello!")
}

// With parameters, no return value
func greet(name string) {
    fmt.Printf("Hello, %s!\n", name)
}

// With parameters and return value
func add(a int, b int) int {
    return a + b
}

// Multiple parameters (same type can be merged)
func addMulti(a, b, c int) int {
    return a + b + c
}
```

### 2. Multiple Return Values ⭐

Go functions can return multiple values:

```go
// Return two values
func divide(a, b int) (int, int) {
    quotient := a / b
    remainder := a % b
    return quotient, remainder
}

// Call
q, r := divide(10, 3)
fmt.Printf("Quotient: %d, Remainder: %d\n", q, r)

// Common pattern: return value + error
func getUser(id int) (User, error) {
    // ...
}

user, err := getUser(1)
if err != nil {
    // Handle error
}
```

### 3. Named Return Values

Return values can be named and treated as variables:

```go
func getPerson() (name string, age int) {
    name = "John"
    age = 25
    return  // Automatically returns name and age
}

// Equivalent to
func getPerson2() (string, int) {
    name := "John"
    age := 25
    return name, age
}
```

**Benefits:** Clearer code, especially with many return values.

**Note:** Don't overuse; simple functions can return directly.

## 🔁 Variadic Parameters

Functions can accept variable number of arguments:

```go
// Variadic parameter must be the last one
func sum(nums ...int) int {
    total := 0
    for _, num := range nums {
        total += num
    }
    return total
}

// Call
fmt.Println(sum(1, 2, 3))       // 6
fmt.Println(sum(1, 2, 3, 4, 5)) // 15

// Pass slice
nums := []int{1, 2, 3, 4}
fmt.Println(sum(nums...))  // Unpack slice
```

**Real-world applications:**

```go
// Format strings
fmt.Printf("Name: %s, Age: %d", name, age)

// Logging
log.Println("Error occurred:", err, "at", time.Now())
```

## 🎯 Functions as Values

In Go, functions are first-class citizens:

```go
// Define function type
type Operation func(int, int) int

// Assign function
var add Operation = func(a, b int) int {
    return a + b
}

// Pass as parameter
func calculate(a, b int, op Operation) int {
    return op(a, b)
}

result := calculate(10, 5, add)
fmt.Println(result)  // 15

// Return as value
func getOperation(op string) Operation {
    switch op {
    case "add":
        return func(a, b int) int { return a + b }
    case "mul":
        return func(a, b int) int { return a * b }
    default:
        return nil
    }
}
```

## 🔒 Anonymous Functions and Closures

### 1. Anonymous Functions

Functions without names:

```go
// Define and execute immediately
func() {
    fmt.Println("Hello!")
}()

// With parameters
func(name string) {
    fmt.Printf("Hello, %s!\n", name)
}("John")

// Assign to variable
add := func(a, b int) int {
    return a + b
}
fmt.Println(add(2, 3))  // 5
```

### 2. Closures ⭐

Closures can capture external variables:

```go
func makeCounter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

counter := makeCounter()
fmt.Println(counter())  // 1
fmt.Println(counter())  // 2
fmt.Println(counter())  // 3
```

**Key point:** The inner function captures and maintains access to `count`.

## ⚡ defer Statement

`defer` delays function execution until the surrounding function returns:

```go
func readFile(filename string) error {
    file, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer file.Close()  // Execute when function returns
    
    // Process file...
    return nil
}
```

**Common uses:**
- Resource cleanup (close files, unlock mutexes)
- Panic recovery
- Timing functions

**Execution order:** LIFO (Last In, First Out)

```go
func main() {
    defer fmt.Println("1")
    defer fmt.Println("2")
    defer fmt.Println("3")
    fmt.Println("Done")
}
// Output: Done, 3, 2, 1
```

## 💡 Best Practices

### 1. Keep functions small

```go
// ✅ Good: Single responsibility
func validateEmail(email string) error {
    if !strings.Contains(email, "@") {
        return errors.New("invalid email")
    }
    return nil
}

// ❌ Bad: Too many responsibilities
func processUser(data []byte) {
    // Validate, parse, save, send email... too much!
}
```

### 2. Use meaningful names

```go
// ✅ Good
func calculateTotalPrice(items []Item) float64

// ❌ Bad
func calc(items []Item) float64
```

### 3. Document exported functions

```go
// Add adds two integers and returns the sum.
func Add(a, b int) int {
    return a + b
}
```

## 🎯 Summary

| Feature | Syntax | Use Case |
|---------|--------|----------|
| Multiple returns | `(int, int)` | Value + error |
| Named returns | `(name string)` | Complex returns |
| Variadic | `nums ...int` | Variable args |
| Function values | `func(int) int` | Callbacks, strategies |
| Closures | Capturing vars | Stateful functions |
| defer | `defer cleanup()` | Resource cleanup |
