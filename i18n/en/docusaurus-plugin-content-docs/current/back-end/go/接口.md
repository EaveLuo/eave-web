# Interfaces - The Soul of Go

Interfaces are one of Go's most powerful features. They define behavior specifications without caring about specific implementations.

## 📐 Interface Definition

```go
// Define interface
type Speaker interface {
    Speak() string
}

// Implement interface (no declaration needed!)
type Person struct {
    Name string
}

func (p Person) Speak() string {
    return "I am " + p.Name
}

// Person automatically implements Speaker interface
```

## 🎯 Implicit Implementation

Go interfaces are **implicitly** implemented:

```go
type Writer interface {
    Write([]byte) (int, error)
}

type File struct{}

func (f File) Write(p []byte) (int, error) {
    // Implementation
    return len(p), nil
}

// File automatically implements Writer - no "implements" keyword!
```

## 💡 Interface Values

An interface value holds:
- **Dynamic type**: The concrete type
- **Dynamic value**: The actual value

```go
var s Speaker = Person{Name: "John"}

// s has:
// - Dynamic type: Person
// - Dynamic value: {Name: "John"}
```

## 🔍 Type Assertions

Extract concrete type from interface:

```go
var i interface{} = "hello"

// Method 1: Direct assertion (panics if wrong)
s := i.(string)  // "hello"

// Method 2: Safe assertion with ok check
s, ok := i.(string)  // "hello", true
n, ok := i.(int)     // 0, false

// Method 3: Type switch
switch v := i.(type) {
case string:
    fmt.Println("String:", v)
case int:
    fmt.Println("Int:", v)
default:
    fmt.Println("Unknown type")
}
```

## 🏗️ Common Standard Interfaces

### io.Reader / io.Writer

```go
type Reader interface {
    Read(p []byte) (n int, err error)
}

type Writer interface {
    Write(p []byte) (n int, err error)
}

// Anything that can be read/written
func copy(dst Writer, src Reader) (int64, error)
```

### Stringer

```go
type Stringer interface {
    String() string
}

// Implement for custom formatting
type Person struct {
    Name string
    Age  int
}

func (p Person) String() string {
    return fmt.Sprintf("%s (%d years old)", p.Name, p.Age)
}
```

## ⚠️ Nil Interface Trap

```go
var p *Person = nil
var s Speaker = p  // s is NOT nil!

fmt.Println(s == nil)  // false
// s has: type=*Person, value=nil
```

**Rule:** Interface is nil only when both type and value are nil.

## 💡 Best Practices

### 1. Small interfaces are better

```go
// ✅ Good: Focused and reusable
type Reader interface {
    Read(p []byte) (n int, err error)
}

// ❌ Bad: Too many methods
type BigInterface interface {
    Read()
    Write()
    Close()
    Open()
    Seek()
    // ... too many
}
```

### 2. Accept interfaces, return structs

```go
// ✅ Good: Flexible input
func process(r io.Reader) error

// ✅ Good: Concrete output
func createUser(name string) (*User, error)
```

### 3. Don't export unnecessary interfaces

```go
// Internal package - no need to export
package internal

type processor interface {
    process(data []byte) error
}
```

## 📊 Summary

| Feature | Description |
|---------|-------------|
| Implementation | Implicit (duck typing) |
| Empty interface | `interface{}` - any type |
| Type assertion | `i.(Type)` or `i.(Type)` with ok |
| Type switch | `switch v := i.(type)` |

**Remember:** In Go, we say "If it walks like a duck and quacks like a duck, it's a duck."
