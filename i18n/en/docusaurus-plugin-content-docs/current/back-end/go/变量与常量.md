# Variables and Constants - Go's Way of Declaration

In Go, there are multiple ways to declare variables and constants. Let's explore Go's unique "Way of Declaration".

## Package Variable Declaration

### 1. Standard Declaration

```go
var age int
age = 25

var name string
name = "John"

var salary float64
salary = 9999.99
```

**Syntax:** `var variableName type`

**Characteristics:**
- Type comes **after** the variable name (opposite of C/Java)
- Uninitialized variables have **zero values**
  - `int` → `0`
  - `string` → `""`
  - `bool` → `false`
  - pointers → `nil`

### 2. Declaration with Initialization

```go
var age int = 25
var name string = "John"
var salary float64 = 9999.99
```

### 3. Type Inference (Recommended) Sparkles

```go
var age = 25        // Automatically inferred as int
var name = "John"   // Automatically inferred as string
var salary = 9999.99 // Automatically inferred as float64
```

The Go compiler is smart enough to infer types from the right-hand side!

### 4. Short Declaration (Most Common) Star

```go
age := 25
name := "John"
salary := 9999.99
```

**This is the most commonly used declaration style in Go!**

**Characteristics:**
- Concise and clear
- Automatic type inference
- **Can only be used inside functions**

### 5. Batch Declaration

```go
// Standard way
var (
    name string
    age  int
    salary float64
)

// With initialization
var (
    name = "John"
    age  = 25
    salary = 9999.99
)

// Short declaration (multiple variables)
name, age, salary := "John", 25, 9999.99
```

## Target Practical Examples

```go
package main

import "fmt"

func main() {
    // Method 1: Standard declaration
    var age int
    age = 25
    fmt.Println("Age:", age)
    
    // Method 2: With initialization
    var name string = "John"
    fmt.Println("Name:", name)
    
    // Method 3: Type inference
    var city = "Beijing"
    fmt.Println("City:", city)
    
    // Method 4: Short declaration (recommended!)
    salary := 9999.99
    fmt.Println("Salary:", salary)
    
    // Method 5: Batch declaration
    var (
        level   = 1
        exp     = 0
        isAlive = true
    )
    fmt.Printf("Level: %d, Exp: %d, Alive: %v\n", level, exp, isAlive)
}
```

## ArrowsClockwise Variable Assignment and Swapping

### Basic Assignment

```go
age := 25
age = 26  // Reassignment
```

### Magic Swap

```go
a := 10
b := 20

// Go-style swap (no temporary variable needed!)
a, b = b, a

fmt.Println(a, b)  // Output: 20 10
```

**Principle:** Go evaluates all right-hand side values simultaneously, then assigns to the left.

## Lock Constants

Once defined, constants cannot be modified.

### 1. Basic Usage

```go
const pi = 3.14159
const maxUsers = 1000
const appName = "MyApp"
```

### 2. Batch Declaration

```go
const (
    Sunday    = 0
    Monday    = 1
    Tuesday   = 2
    Wednesday = 3
    Thursday  = 4
    Friday    = 5
    Saturday  = 6
)
```

### 3. Iota Enumerator Star

`iota` is Go's enumeration counter, automatically incrementing from 0:

```go
const (
    Sunday = iota  // 0
    Monday         // 1
    Tuesday        // 2
    Wednesday      // 3
    Thursday       // 4
    Friday         // 5
    Saturday       // 6
)
```

**Cooler usage:**

```go
const (
    _ = iota             // Skip 0
    KB = 1 << (10 * iota) // 1 << 10 = 1024
    MB                    // 1 << 20 = 1048576
    GB                    // 1 << 30
    TB                    // 1 << 40
)

fmt.Println(KB)  // 1024
fmt.Println(MB)  // 1048576
fmt.Println(GB)  // 1073741824
```

### 4. Constant Expressions

```go
const (
    width  = 1920
    height = 1080
    area   = width * height  // Constant expression, computed at compile time
)
```

## BalanceScale Variables vs Constants

| Feature | Variables | Constants |
|---------|-----------|-----------|
| Modifiable | WhiteCheckMark Yes | CrossMark No |
| Declaration Keyword | `var` / `:=` | `const` |
| Type Inference | WhiteCheckMark Supported | WhiteCheckMark Supported |
| Scope | Any | Any |
| Compile-time Evaluation | CrossMark No | WhiteCheckMark Yes |

## ArtistPalette Naming Conventions

### 1. Visibility Rules

Go controls visibility through **first letter capitalization**:

```go
// Capitalized = Exported (public)
var Name = "John"  // Accessible from other packages

// Lowercase = Unexported (private)
var name = "Jane"  // Only accessible in current package
```

**This is Go's unique design! No `public`/`private` keywords needed.**

### 2. Naming Recommendations

```go
// WhiteCheckMark Good naming
var userName string
var userID int
var isValid bool
var totalAmount float64

// CrossMark Bad naming
var a int           // Too short, meaningless
var theName string  // Redundant
var flag bool       // Not specific enough
```

### 3. Short Naming Scenarios

```go
// Loop variables
for i := 0; i < 10; i++ {
    // i is clear enough
}

// Error return values
err := doSomething()
if err != nil {
    // err is conventional naming
}

// Receivers (in methods)
func (u *User) GetName() string {
    // u is common naming
}
```

## Bug Common Mistakes

### 1. Redeclaration

```go
// CrossMark Error
age := 25
age := 26  // Compile error: no new variables

// WhiteCheckMark Correct
age := 25
age = 26   // Reassignment, no :=
```

### 2. Unused Variables

```go
func main() {
    age := 25  // CrossMark Compile error: declared and not used
    fmt.Println("Hello")
}
```

**Go compiler checks for unused variables!**

**Solutions:**
```go
// Temporarily comment out
// age := 25

// Or use blank identifier
_ = age
```

### 3. Constant Reassignment

```go
const pi = 3.14
pi = 3.14159  // CrossMark Compile error: cannot assign to pi
```

## LightBulb Best Practices

### 1. Prefer Short Declarations

```go
// WhiteCheckMark Recommended
name := "John"
age := 25

// Warning unless you need to specify type
var data []byte = make([]byte, 1024)
```

### 2. Batch Declare Related Variables

```go
// WhiteCheckMark Clear
var (
    host = "localhost"
    port = 8080
    path = "/api"
)

// CrossMark Scattered
var host = "localhost"
var port = 8080
var path = "/api"
```

### 3. Use Uppercase for Constants

```go
// WhiteCheckMark Recommended
const (
    MaxRetry = 3
    Timeout  = 30
)

// Warning not recommended
const (
    maxRetry = 3
    timeout  = 30
)
```

### 4. Meaningful Names

```go
// WhiteCheckMark Clear
daysInWeek := 7
maxConnection := 100

// CrossMark Vague
d := 7
m := 100
```

## Target Practice Exercise

```go
package main

import "fmt"

func main() {
    // 1. Declare personal info
    name := "John"
    age := 25
    city := "Beijing"
    
    // 2. Declare work info
    var company string = "Tech Corp"
    var salary float64 = 15000
    level := 5
    
    // 3. Use constants
    const taxRate = 0.1
    afterTax := salary * (1 - taxRate)
    
    // 4. Swap values
    a, b := 10, 20
    a, b = b, a
    
    // 5. Output
    fmt.Printf("%s, %d years old, lives in %s\n", name, age, city)
    fmt.Printf("Works at %s, level %d, after-tax salary %.2f\n", 
               company, level, afterTax)
    fmt.Printf("After swap: a=%d, b=%d\n", a, b)
}
```
