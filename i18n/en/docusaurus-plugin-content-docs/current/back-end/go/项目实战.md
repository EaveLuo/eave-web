# Project Practice - Real-World Applications

Apply what you've learned to real projects.

## 🎯 Project Ideas

### 1. CLI Tool

Build a command-line tool with flags and subcommands:

```go
// todo-cli
// Commands: add, list, done, delete

type Task struct {
    ID      int
    Title   string
    Done    bool
    Created time.Time
}
```

### 2. REST API Server

Create a complete HTTP API:

```go
// User management API
// Endpoints:
// GET    /users
// POST   /users
// GET    /users/:id
// PUT    /users/:id
// DELETE /users/:id
```

## 📁 Project Structure

```
myproject/
├── cmd/
│   └── myapp/
│       └── main.go
├── internal/
│   ├── config/
│   ├── handler/
│   └── service/
├── pkg/
│   └── utils/
├── go.mod
└── README.md
```

## 💡 Best Practices

1. **Use standard layout** - Follow Go project conventions
2. **Write tests** - Aim for good coverage
3. **Document code** - Clear comments and README
4. **Handle errors** - Never ignore errors
5. **Use context** - For cancellation and timeouts
